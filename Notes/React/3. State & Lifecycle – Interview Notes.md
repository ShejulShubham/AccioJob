
## **1. What is state in React?**

**State** is a built-in object that stores **dynamic, changeable data** of a component.

- Unlike props, **state can change over time**.
    
- When state updates, the component **re-renders automatically**.
    

### Example (Functional Component):

```jsx
const [count, setCount] = useState(0);
```

---

## **2. How is state different from props?**

|Feature|State|Props|
|---|---|---|
|Modifiable|Yes|No (read-only)|
|Managed by|Component itself|Parent component|
|Used for|Dynamic data|Passing data|
|Causes re-render|Yes|Yes|
|Access in functional|useState()|Function argument|

**In short:**

- Props = external, passed from parent
    
- State = internal, controlled by the component

## **3. How do you update state in a class component?**

In class components, state is updated using **`this.setState()`**.

### Example:

```jsx
class Counter extends React.Component {
  state = { count: 0 };

  increment = () => {
    this.setState({ count: this.state.count + 1 });
  };

  render() {
    return <button onClick={this.increment}>{this.state.count}</button>;
  }
}
```

### Important rules:

- Never modify state directly → ❌ `this.state.count = 1`
    
- Use `setState()` because it triggers a re-render
    
- `setState()` is asynchronous — use callback if needed

## **4. What are lifecycle methods in React class components?**

Lifecycle methods are methods that run at **specific points** in a component’s life:

### **Mounting (when component is created)**

- `constructor()`
    
- `componentDidMount()`
    

### **Updating (when props/state changes)**

- `componentDidUpdate()`
    

### **Unmounting (when component is removed)**

- `componentWillUnmount()`
    

### **Error handling**

- `componentDidCatch()`
    

They help perform tasks like API calls, timers, subscriptions, cleanup, etc.

---

## **5. What are `componentDidMount()` and `componentWillUnmount()` used for?**

### **🟢 `componentDidMount()`**

Runs **once** after the component first renders.

Used for:

- Fetching data (API calls)
    
- Adding event listeners
    
- Setting up timers/subscriptions
    

### Example:

```jsx
componentDidMount() {
  console.log("Mounted!");
}
```

---

### **🔴 `componentWillUnmount()`**

Runs right before the component is removed from the DOM.

Used for cleanup:

- Removing event listeners
    
- Clearing timers/intervals
    
- Canceling subscriptions
    

### Example:

```jsx
componentWillUnmount() {
  clearInterval(this.timer);
}
```

## **6. How do you handle side effects in functional components?**

Functional components use the **`useEffect()` hook** for side effects.

### Side effects include:

- API calls
    
- Timers
    
- Subscriptions
    
- DOM manipulation
    

### Basic example:

```jsx
useEffect(() => {
  console.log("Effect running");
}, []);
```

### Cleanup example:

```jsx
useEffect(() => {
  const timer = setInterval(() => console.log("tick"), 1000);

  return () => clearInterval(timer); // cleanup
}, []);
```

### How `useEffect` works:

- First argument → function to run
    
- Second argument → dependency array
    
    - `[]` → runs once (mount)
        
    - `[value]` → runs when `value` changes
        
    - No array → runs on every render
