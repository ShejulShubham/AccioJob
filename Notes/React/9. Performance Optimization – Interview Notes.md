
## **1. What causes re-renders in React?**

A React component re-renders when:

### **A. Its state changes**

```jsx
setCount(count + 1);  // triggers re-render
```

### **B. Its props change**

Parent passes new props → re-render.

### **C. Its parent re-renders**

Child re-renders unless memoized.

### **D. Context value changes**

All consumers re-render.

### **E. Force update**

Using `forceUpdate()` (class) or special patterns.

## **2. How can you optimize rendering performance?**

### **Techniques:**

### ✔ **1. Use React.memo()**

Prevents unnecessary re-renders of functional components.

### ✔ **2. useMemo() & useCallback()**

- Memoize expensive calculations
    
- Memoize functions to avoid re-creation
    

### ✔ **3. Lazy loading components**

Load components only when needed.

### ✔ **4. Code splitting**

Reduce initial JavaScript bundle size.

### ✔ **5. Avoid inline functions & objects**

Because they create new references every render.

### ✔ **6. Avoid unnecessary state**

Keep state minimal and close to where needed.

### ✔ **7. Virtualization (large lists)**

Using libraries like **react-window**.

### ✔ **8. Use key wisely**

Stable keys avoid re-rendering entire lists.

## **3. What is React.memo()?**

`React.memo()` is a higher-order component that **prevents re-renders** if props haven’t changed.

### Example:

```jsx
const Child = React.memo(function Child({ value }) {
  console.log("Rendered");
  return <h1>{value}</h1>;
});
```

### When to use:

- Pure/dumb components
    
- Components that re-render frequently
    
- Components with heavy UI
    

### Note:

- It shallow compares props
    
- Use carefully to avoid over-optimization
    

---

## **4. What does `useMemo` and `useCallback` do?**

### **A. `useMemo` → Memoizes expensive calculations**

Prevents expensive functions from running on every render.

### Example:

```jsx
const expensiveValue = useMemo(() => heavyCalculation(num), [num]);
```

### Use when:

- A calculation is heavy/expensive
    
- Value doesn't need to recompute every render
    

---

### **B. `useCallback` → Memoizes functions**

Prevents function re-creation unless dependencies change.

### Example:

```jsx
const handleClick = useCallback(() => {
  console.log("clicked");
}, []);
```

### Use when:

- Passing functions to memoized children
    
- Preventing unnecessary re-renders
    
- Stabilizing event handlers or callbacks
    

---

## **5. How do you lazy-load components?**

Use `React.lazy()` and `Suspense` to load components **only when needed**.

### Example:

```jsx
const Profile = React.lazy(() => import("./Profile"));

function App() {
  return (
    <Suspense fallback={<div>Loading...</div>}>
      <Profile />
    </Suspense>
  );
}
```

### Benefits:

- Reduces initial bundle size
    
- Improves load time
    
- Better performance for large apps
