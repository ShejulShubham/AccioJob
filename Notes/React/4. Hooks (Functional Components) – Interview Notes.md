
## **1. What are React Hooks? Why were they introduced?**

### **React Hooks**

Hooks are **functions** that let functional components use:

- **State**
    
- **Lifecycle methods**
    
- **Context**
    
- And more…
    

Hooks make functional components as powerful as class components.

### **Why they were introduced**

- To remove the need for class components
    
- To reuse stateful logic without complex patterns like HOCs or render props
    
- To make code simpler and more readable
    
- To make components smaller, cleaner, and easier to test
    

Example:

```jsx
const [count, setCount] = useState(0);
```

## **2. What does the `useState` hook do?**

`useState` allows functional components to have **state**.

### Syntax:

```jsx
const [state, setState] = useState(initialValue);
```

### Example:

```jsx
const [name, setName] = useState("Shubham");
```

### Key points:

- State updates cause re-renders
    
- `setState` is asynchronous
    
- You **must** use the setter function (not update directly)

## **3. What is `useEffect` used for?**

`useEffect` handles **side effects**, such as:

- API calls
    
- Subscriptions
    
- Event listeners
    
- Timers
    
- Logging
    
- Updating the document title
    

### Syntax:

```jsx
useEffect(() => {
  // side effect logic
}, [dependencies]);
```

---

## **4. What is the difference between `useEffect` and lifecycle methods?**

|Class Components|Functional Components|
|---|---|
|`componentDidMount()`|`useEffect(..., [])`|
|`componentDidUpdate()`|`useEffect(..., [value])`|
|`componentWillUnmount()`|Cleanup inside `useEffect` return|

### Example of cleanup:

```jsx
useEffect(() => {
  const timer = setInterval(() => console.log("tick"), 1000);

  return () => clearInterval(timer); // cleanup
}, []);
```

### Main difference:

- Hooks **combine multiple lifecycle behaviors in one place**
    
- Hooks are **function-based**, not class-based

## **5. What are rules of hooks?**

React enforces these rules so hooks work correctly:

### ✔ Rule 1: Only call hooks at the **top level**

- ❌ Not inside loops
    
- ❌ Not inside conditions
    
- ❌ Not inside nested functions
    

### ✔ Rule 2: Only call hooks inside **React components or custom hooks**

- Not in normal JS functions
    

### ✔ Rule 3: Component name must start with **capital letter**

These rules ensure React can correctly track hook calls and state.

## **6. What is `useRef` used for?**

`useRef` creates a **mutable reference** that **doesn’t cause re-renders**.

### Uses:

- Accessing DOM elements
    
- Storing values between renders
    
- Holding timeout IDs, previous values, or counters
    

### Example: DOM reference

```jsx
const inputRef = useRef();

function handleFocus() {
  inputRef.current.focus();
}

<input ref={inputRef} />;
```

### Example: Storing previous value

```jsx
const prevCount = useRef();
prevCount.current = count;
```

## **7. Explain the purpose of `useContext`.**

`useContext` allows components to **consume values** from React’s **Context API** without prop drilling.

### Used for:

- Sharing global state
    
- Sharing theme, user data, language settings
    
- Avoiding passing props through many nested components
    

### Example:

```jsx
const ThemeContext = createContext();

function App() {
  return (
    <ThemeContext.Provider value="dark">
      <Home />
    </ThemeContext.Provider>
  );
}

function Home() {
  const theme = useContext(ThemeContext);
  return <p>Theme: {theme}</p>;
}
```

## **8. What is `useReducer`? When should you use it?**

`useReducer` is an alternative to `useState` for managing **complex state logic**.

### Syntax:

```jsx
const [state, dispatch] = useReducer(reducer, initialState);
```

### Reducer function:

```jsx
function reducer(state, action) {
  switch (action.type) {
    case "increment":
      return { count: state.count + 1 };
    default:
      return state;
  }
}
```

### Example:

```jsx
const [state, dispatch] = useReducer(reducer, { count: 0 });
```

### When to use useReducer?

Use when:

- State has **multiple sub-values**
    
- You need **structured** updates
    
- State transitions are complex
    
- You want Redux-like behavior
    

Perfect for:

- Forms
    
- Complex UI interactions
    
- Large components managing many values
