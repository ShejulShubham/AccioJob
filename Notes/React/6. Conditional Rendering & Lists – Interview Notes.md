
## **1. How can you conditionally render components?**

React allows conditional rendering using standard JavaScript techniques:

### **A. if/else**

```jsx
if (isLoggedIn) {
  return <Dashboard />;
}
return <Login />;
```

### **B. Ternary operator (most commonly used)**

```jsx
{isDark ? <DarkMode /> : <LightMode />}
```

### **C. Logical AND (&&)**

Useful for rendering something only when a condition is true.

```jsx
{isAdmin && <AdminPanel />}
```

### **D. Switch statements**

Useful for multiple conditions.

```jsx
switch (status) {
  case "loading": return <Loading />;
  case "error": return <Error />;
  default: return <Content />;
}
```

## **2. How do you render a list of elements in React?**

Use `.map()` to iterate through an array and return JSX elements.

### Example:

```jsx
const users = ["Shubham", "Amit", "Raj"];

return (
  <ul>
    {users.map((user, index) => (
      <li key={index}>{user}</li>
    ))}
  </ul>
);
```

### Note:

Each element **must** have a **unique key**.

---

## **3. Why should every list item have a unique key?**

React uses keys to **identify list items**, helping its **diffing algorithm** update only what has changed.

### Keys help React:

- Track which items are added/removed/reordered
    
- Avoid unnecessary re-renders
    
- Improve performance
    
- Prevent bugs with component state inside lists
    

### Important:

- Keys should be **unique**, **stable**, and **consistent**
    
- Avoid using **index as key** when the list can change
    

Better key example:

```jsx
{users.map(user => (
  <li key={user.id}>{user.name}</li>
))}
```

## **4. What are fragments in React?**

**Fragments** allow you to group multiple elements without adding an extra DOM node (like a `<div>`).

### Why use them?

- Cleaner HTML structure
    
- Avoid unnecessary wrappers
    
- Better for lists, tables, layouts
    

### Example:

```jsx
<>
  <h1>Title</h1>
  <p>Description</p>
</>
```

### Alternative syntax:

```jsx
<React.Fragment>
  <h1>Title</h1>
  <p>Description</p>
</React.Fragment>
```

Useful when:

- Rendering siblings without extra wrapping
    
- Avoiding CSS layout issues caused by extra divs
