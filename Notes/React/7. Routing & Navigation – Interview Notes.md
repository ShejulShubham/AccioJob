
## **1. What is React Router?**

**React Router** is the **standard routing library** used to create **single-page application (SPA)** navigation in React.

### What it provides:

- Client-side routing
    
- Navigation without reloading the page
    
- Dynamic routes
    
- Nested routes
    
- URL-based state
    

It allows browsers to change the URL **without full page refresh**, giving a smooth app-like experience.

---

## **2. How do you set up routes in a React app?**

Install:

```bash
npm install react-router-dom
```

Basic setup with **BrowserRouter**, **Routes**, and **Route**:

### Example:

```jsx
import { BrowserRouter, Routes, Route } from "react-router-dom";

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </BrowserRouter>
  );
}
```

### Key components:

- **BrowserRouter** → enables routing
    
- **Routes** → wrapper for route definitions
    
- **Route** → maps a URL path to a component

## **3. What is the difference between `<Link>` and `<NavLink>`?**

### **`<Link>`**

- Used for simple navigation
    
- Does **not** apply any styling automatically
    
- Basic link component
    

```jsx
<Link to="/home">Home</Link>
```

---

### **`<NavLink>`**

- Used for navigation **with active styling**
    
- Automatically adds an **"active"** class when the link matches the current URL
    
- Good for navigation menus or tabs
    

```jsx
<NavLink 
  to="/home"
  className={({ isActive }) => isActive ? "active-link" : ""}
>
  Home
</NavLink>
```

### Summary:

|Feature|Link|NavLink|
|---|---|---|
|Basic navigation|✔|✔|
|Active styling|❌|✔|
|For menus|Moderate|Best|

## **4. How do you navigate programmatically in React Router?**

Use the `useNavigate()` hook.

### Example:

```jsx
import { useNavigate } from "react-router-dom";

function Login() {
  const navigate = useNavigate();

  const handleLogin = () => {
    // after successful login
    navigate("/dashboard");
  };

  return <button onClick={handleLogin}>Login</button>;
}
```

You can also navigate backward/forward:

```jsx
navigate(-1); // go back
navigate(1);  // go forward
```

## **5. What is a dynamic route?**

A **dynamic route** contains **parameters** that change based on user action or data — such as user profiles, product pages, or blog posts.

### Example route:

```jsx
<Route path="/user/:id" element={<UserProfile />} />
```

### Accessing route parameters:

```jsx
import { useParams } from "react-router-dom";

function UserProfile() {
  const { id } = useParams(); 
  return <h1>User ID: {id}</h1>;
}
```

### Use cases:

- `/products/101`
    
- `/blog/react-hooks-explained`
    
- `/user/shubham123`
    

Dynamic routes allow reusable components for multiple pages.
