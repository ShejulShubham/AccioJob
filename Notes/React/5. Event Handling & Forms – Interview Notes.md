
## **1. How do you handle events in React?**

React uses **camelCase event names** and **functions** to handle events.

### Example:

```jsx
function handleClick() {
  console.log("Button clicked!");
}

<button onClick={handleClick}>Click Me</button>
```

### Key differences from HTML:

- HTML → `onclick="handler()"`
    
- React → `onClick={handler}` (no quotes, no function call)
    

---

## **2. What are synthetic events?**

**Synthetic events** are React’s **cross-browser wrapper** around native browser events.

### Why React uses synthetic events:

- Ensures consistent behavior across browsers
    
- Standardized event handling
    
- Better performance through event delegation
    

### Example:

`onClick`, `onChange`, `onSubmit`, etc. are all synthetic events.

## **3. How do you handle form inputs in React?**

You use **state** to store form values and **onChange** handlers to update state.

### Example:

```jsx
function Form() {
  const [name, setName] = useState("");

  return (
    <input 
      value={name}
      onChange={(e) => setName(e.target.value)}
    />
  );
}
```

---

## **4. What is a controlled component?**

A **controlled component** is an input field whose value is **fully controlled by React state**.

### Example:

```jsx
const [email, setEmail] = useState("");

<input 
  value={email} 
  onChange={(e) => setEmail(e.target.value)} 
/>
```

### Characteristics:

- Source of truth = React state
    
- You can validate or modify input easily
    
- Most common pattern in React apps
    

---

## **5. What is an uncontrolled component?**

An **uncontrolled component** stores its own internal state inside the DOM—not in React state.

You access values using **refs** instead of state.

### Example:

```jsx
function Form() {
  const inputRef = useRef();

  const handleSubmit = () => {
    console.log(inputRef.current.value);
  };

  return <input ref={inputRef} />;
}
```

### When to use?

- Integrating with non-React libraries
    
- Very simple forms
    
- When you don’t need to control state on each keystroke

## **6. How do you prevent default form submission in React?**

Use `event.preventDefault()` inside the form’s submit handler.

### Example:

```jsx
function Form() {
  const handleSubmit = (e) => {
    e.preventDefault();
    console.log("Form submitted");
  };

  return (
    <form onSubmit={handleSubmit}>
      <button type="submit">Submit</button>
    </form>
  );
}
```

### Why needed?

- React single-page apps don't reload pages
    
- You want to control what happens after submission (API call, validation, etc.)
