## **1. What is React.js and why is it used?**

**React.js** is a **JavaScript library** for building **user interfaces**, especially **single-page applications (SPAs)**.  
Created by Facebook, it allows developers to build UI in a component-based structure.

### **Why it is used**

- Fast UI rendering using **Virtual DOM**
    
- Reusable **components**
    
- Easy to manage state and UI updates
    
- Strong ecosystem and community support
    

---

## **2. What are the main features of React?**

- **JSX** → JavaScript syntax extension for writing UI in HTML-like code
    
- **Components** → Reusable building blocks
    
- **Virtual DOM** → Efficient UI updates
    
- **One-way data flow** → Predictable data management
    
- **Hooks** → Add state and lifecycle logic to functional components
    
- **Declarative UI** → Describe what UI should look like, React updates the DOM

## **3. What is the difference between a library and a framework?**

|Library|Framework|
|---|---|
|You call the library.|Framework calls your code (Inversion of control).|
|Gives tools to use as needed.|Defines structure and rules for the application.|
|More flexible.|More opinionated.|
|Example: **React**|Example: **Angular, Django**|

**React is a library**, not a framework.

## **4. What is the Virtual DOM and how does it work?**

**Virtual DOM (VDOM)** is a lightweight copy of the real DOM stored in memory.

### **How it works**

1. React creates a virtual DOM representation of the UI.
    
2. When the UI changes, a **new VDOM** is created.
    
3. React compares old VDOM and new VDOM using **diffing algorithm**.
    
4. Only the changed parts are updated in the **real DOM** (patching).
    

### **Benefit**

- Faster and more efficient updates
    
- Improved performance for complex UIs

## **5. Explain the concept of JSX.**

**JSX** stands for **JavaScript XML**.  
It allows writing HTML-like syntax inside JavaScript.

### Example:

```jsx
const element = <h1>Hello World!</h1>;
```

### Why useful?

- Clear UI structure inside JS
    
- Compiled to `React.createElement()` by Babel
    
- Makes components readable and maintainable
    

---

## **6. Why do we need a root element in React?**

React applications need a **single DOM root** where the entire component tree is mounted.

Typical snippet:

```html
<div id="root"></div>
```

### Why?

- React manages everything _inside_ this root
    
- Improves rendering control
    
- Avoids conflicts with non-React code

## **7. What are React components?**

Components are **independent, reusable pieces of UI**.

### Types:

- **Functional components**
    
- **Class components**
    

### Example:

```jsx
function Welcome() {
  return <h1>Hello</h1>;
}
```

### Benefits:

- Reusability
    
- Easy to test
    
- Clean structure
    

---

## **8. Differentiate between class components and functional components.**

|Feature|Class Component|Functional Component|
|---|---|---|
|Syntax|ES6 class|JavaScript function|
|State|Uses `this.state`|Uses `useState()`|
|Lifecycle|`componentDidMount`, etc.|`useEffect()`|
|Performance|Slightly slower|Faster & simpler|
|Hooks|❌ Not applicable|✔️ Supported|

### Example Functional Component:

```jsx
function Counter() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}
```

## **9. What is the significance of keys in React?**

**Keys** help React identify which list items changed, added, or removed.

### Why important?

- Improves performance
    
- Avoids re-rendering entire lists
    
- Helps diffing algorithm
    

### Example:

```jsx
{items.map(item => <li key={item.id}>{item.name}</li>)}
```

## **10. How does React handle rendering performance?**

React optimizes performance through:

### **1. Virtual DOM**

Minimizes actual DOM changes.

### **2. Reconciliation**

Only updates changed components.

### **3. Keys**

Helps efficiently update list items.

### **4. Memoization**

- `React.memo()` to prevent unnecessary re-renders
    
- `useMemo()` to memoize values
    
- `useCallback()` to memoize functions
    

### Example:

```jsx
const MemoizedComponent = React.memo(MyComponent);
```

### **5. Batching Updates**

Multiple state updates grouped into one render.

### **6. Lazy loading**

Using `React.lazy()` for code-splitting.
