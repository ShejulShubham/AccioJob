### **1. What is React?**

**Answer:**

React is a JavaScript library used to build user interfaces, especially single-page applications. It focuses on building reusable UI components and efficiently updating the UI using a virtual DOM.

---

### **2. Why is React called a library and not a framework?**

**Answer:**

React only handles the UI layer of an application. It doesn’t enforce routing, state management, or data handling patterns. Developers choose additional libraries as needed, which makes React flexible.

---

### **3. What are the main features of React?**

**Answer:**

- Component-based architecture
- Virtual DOM
- Declarative UI
- One-way data flow
- Hooks for state and lifecycle management

---

### **4. What is JSX?**

**Answer:**

JSX is a syntax extension for JavaScript that allows writing HTML-like code inside JavaScript. It gets transpiled into `React.createElement()` calls by tools like Babel.

---

### **5. Can React work without JSX?**

**Answer:**

Yes. JSX is optional. React can be written using `React.createElement()`, but JSX improves readability and developer experience.

---

### **6. What is a component in React?**

**Answer:**

A component is a reusable piece of UI. It can be a function or a class that returns JSX and optionally manages its own state and logic.

---

### **7. What are the types of components in React?**

**Answer:**

- Functional Components
- Class Components

Modern React primarily uses functional components with hooks.

---

### **8. What is the difference between functional and class components?**

**Answer:**

Functional components are simple JavaScript functions and use hooks for state and lifecycle.

Class components use ES6 classes and lifecycle methods. Functional components are preferred today.

---

### **9. What is props in React?**

**Answer:**

Props (properties) are read-only inputs passed from a parent component to a child component. They help make components reusable and dynamic.

---

### **10. What is state in React?**

**Answer:**

State is a built-in object used to store data that can change over time. When state changes, the component re-renders to reflect the updated data.
