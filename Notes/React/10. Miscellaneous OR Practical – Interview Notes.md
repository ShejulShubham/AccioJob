
## **1. What is create-react-app?**

**Create React App (CRA)** is a command-line tool that bootstraps a React project with a ready-to-use setup.

### It provides:

- Webpack configuration
    
- Babel setup
    
- Development server
    
- Build pipeline
    
- Hot reloading
    

### Command:

```bash
npx create-react-app my-app
```

CRA saves developers from manual setup and provides a production-ready environment out of the box.

## **2. How do you import and export components?**

### **Exporting a component**

```jsx
// MyComponent.js
export default MyComponent;
```

or

```jsx
export const Header = () => { ... };
```

### **Importing a component**

```jsx
import MyComponent from "./MyComponent";
```

or

```jsx
import { Header } from "./Header";
```

---

## **3. What is the difference between default and named exports?**

### **Default Export**

- Only one default export per file
    
- Imported with **any name**
    

```jsx
export default Button;
import MyButton from "./Button";
```

---

### **Named Export**

- Can have multiple
    
- Must use **exact same name** when importing
    

```jsx
export const Header = () => {};
export const Footer = () => {};

import { Header, Footer } from "./Layout";
```

---

### Summary:

|Feature|Default Export|Named Export|
|---|---|---|
|Count|One per file|Many per file|
|Import name|Can rename|Must match|
|Use case|Main component|Utilities, multiple components|

## **4. How do you deploy a React app?**

Multiple platforms exist depending on the project type.

### **Basic steps for CRA app:**

1. Build the project:
    

```bash
npm run build
```

2. Deploy the `build/` folder to a hosting provider like:
    

- **Netlify**
    
- **Vercel**
    
- **GitHub Pages**
    
- **Firebase Hosting**
    
- **Render**
    
- **AWS S3 + CloudFront**
    

### Example (Netlify):

- Drag and drop `build` folder
    
- Or connect GitHub repo and enable auto-deploy

## **6. What are higher-order components (HOC)?**

A **Higher-Order Component** is a function that takes a component and returns a new component with added functionality.

### Example:

```jsx
function withLogger(Component) {
  return function Enhanced(props) {
    console.log("Component rendered");
    return <Component {...props} />;
  };
}
```

### Use cases:

- Authentication wrappers
    
- Logging
    
- Theming
    
- Reusing logic before React Hooks existed
    

Not used as often today because **custom hooks replaced most HOCs**.

## **7. What are portals in React?**

Portals allow you to **render a component outside its parent DOM hierarchy**, while keeping it in the **React virtual tree**.

### Example:

```jsx
ReactDOM.createPortal(
  <Modal />,
  document.getElementById("modal-root")
);
```

### Use cases:

- Modals
    
- Tooltips
    
- Dropdowns
    
- Popovers
    

They ensure proper z-index behavior and accessibility.

## **8. What is the difference between React and React Native?**

|Feature|React|React Native|
|---|---|---|
|Purpose|Build web apps|Build mobile apps|
|Output|HTML, CSS, JS|Native mobile UI components|
|Styling|CSS|Uses stylesheets similar to CSS but with JS (`StyleSheet`)|
|Rendering|DOM|Native mobile components (UIView, TextView)|
|Navigation|react-router|react-navigation|
|Platform|Browser|iOS, Android|

### Summary:

- **React** → Web development
    
- **React Native** → Mobile app development using JavaScript + native components
