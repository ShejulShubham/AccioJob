
## **1. What are props in React?**

**Props (properties)** are **read-only inputs** passed from a **parent component to a child component**.

They allow components to be:

- **Reusable**
    
- **Dynamic**
    
- **Configurable**
    

### Example:

```jsx
function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}
```

Here, `name` is a **prop**.

---

## **2. How do you pass data from parent to child components?**

You pass data via **props**.

### Parent Component:

```jsx
<Welcome name="Shubham" />
```

### Child Component:

```jsx
function Welcome({ name }) {
  return <h1>Hello, {name}</h1>;
}
```

---

## **3. Can data be passed from child to parent in React? If yes, how?**

Yes.  
React uses **unidirectional data flow**, so child-to-parent communication is done by:

✅ **Passing a callback function from parent to child**,  
and the child calls that function with data.

### Example:

**Parent:**

```jsx
function Parent() {
  const handleData = (value) => {
    console.log("Received from child:", value);
  };

  return <Child sendData={handleData} />;
}
```

**Child:**

```jsx
function Child({ sendData }) {
  return (
    <button onClick={() => sendData("Hello parent!")}>
      Send Data
    </button>
  );
}
```

## **4. What are default props and how are they used?**

**Default props** define fallback values for props when they are **not provided** by the parent.

### Example:

```jsx
function Button({ label }) {
  return <button>{label}</button>;
}

Button.defaultProps = {
  label: "Click me",
};
```

If `<Button />` is rendered without a `label`, it will show **"Click me"**.

---

## **5. What is prop drilling? How can it be avoided?**

### **Prop drilling**:

When data is passed **multiple levels down the component tree** even when intermediate components **don’t need** that data.

Example:

```
App → Parent → Child → GrandChild
```

Passing props through each layer → **prop drilling**.

### **How to avoid it?**

✔ **React Context API**  
✔ **State management libraries** like Redux, Zustand, MobX  
✔ **Component composition** (lifting logic up, restructuring components)  
✔ **Custom hooks**

## **6. How do you validate props in React?**

React uses **PropTypes** to validate props, useful for catching bugs during development.

### Example:

```jsx
import PropTypes from "prop-types";

function User({ name, age }) {
  return <p>{name} - {age}</p>;
}

User.propTypes = {
  name: PropTypes.string.isRequired,
  age: PropTypes.number,
};
```

### Why use PropTypes?

- Ensures correct data type
    
- Helps in debugging
    
- Improves component reliability
