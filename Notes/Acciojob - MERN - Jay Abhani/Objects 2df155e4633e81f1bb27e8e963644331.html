Objects

Objects in JavaScript

In JavaScript, objects are one of the most fundamental and versatile data types. They allow you to store collections of data and more complex entities in a structured way.

🔹 What is an Object in JavaScript?

An object is a collection of key-value pairs, where each key (also called a property name) is a string (or symbol), and each value can be any data type — number, string, boolean, array, function, or even another object.

🔸 Syntax:

let person = {
  name: "Alice",
  age: 25,
  isStudent: true
};

🔹 Creating Objects

There are several ways to create objects in JavaScript:

1. Object Literal Syntax (Most common)

let car = {
  brand: "Toyota",
  model: "Camry",
  year: 2022
};

2. Using the new Object() syntax

let car = new Object();
car.brand = "Toyota";
car.model = "Camry";

3. Using Constructor Functions

function Person(name, age) {
  this.name = name;
  this.age = age;
}

let person1 = new Person("John", 30);

4. Using Classes (ES6+)

class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }
}

let person1 = new Person("Jane", 28);

🔹 Accessing Object Properties

You can access properties using dot notation or bracket notation.

console.log(car.brand);       // Dot notation
console.log(car["model"]);    // Bracket notation
✅ Bracket notation is useful when the property name is dynamic or not a valid identifier.

🔹 Modifying Object Properties

car.year = 2023;
car["model"] = "Corolla";

🔹 Adding New Properties

car.color = "Red";

🔹 Deleting Properties

delete car.model;

🔹 Nested Objects

Objects can contain other objects:

let user = {
  name: "Alice",
  address: {
    city: "New York",
    zip: "10001"
  }
};

console.log(user.address.city);  // Output: New York

🔹 Object Methods

You can store functions inside objects — these are called methods.

let person = {
  name: "Bob",
  greet: function () {
    return `Hello, I'm ${this.name}`;
  }
};

console.log(person.greet());  // Output: Hello, I'm Bob

🔹 The this Keyword

Within an object method, this refers to the object itself.

let person = {
  name: "Lisa",
  greet() {
    console.log(this.name);  // Refers to 'person.name'
  }
};

🔹 Object Built-in Methods

Object.keys(obj)

Returns an array of keys.

Object.keys(person);  // ["name", "greet"]

Object.values(obj)

Returns an array of values.

Object.values(person);

Object.entries(obj)

Returns an array of [key, value] pairs.

Object.entries(person);

Object.assign(target, ...sources)

Copies properties from source to target object.

let obj1 = { a: 1 };
let obj2 = { b: 2 };
let combined = Object.assign({}, obj1, obj2);
// combined = { a: 1, b: 2 }

Object.freeze(obj)

Makes an object immutable (cannot add, remove, or change properties).

Object.seal(obj)

Allows property modification but prevents adding/removing properties.

🔹 Iterating Over Objects

Using for...in loop:

for (let key in person) {
  console.log(`${key}: ${person[key]}`);
}
for...in iterates over enumerable properties (including inherited).

🔹 Object Destructuring (ES6+)

Allows extracting properties into variables:

let { name, age } = person;

You can also rename:

let { name: personName } = person;

🔹 Comparing Objects

In JavaScript, objects are reference types, so:

let obj1 = { a: 1 };
let obj2 = { a: 1 };
console.log(obj1 === obj2);  // false (different memory reference)

But:

let obj3 = obj1;
console.log(obj1 === obj3);  // true (same reference)

🔹 JSON and Objects

Objects can be converted to JSON using JSON.stringify() and back to objects using JSON.parse().

let jsonString = JSON.stringify(person);
let jsonObject = JSON.parse(jsonString);

🔹 Use Cases of Objects

  • Storing user information
  • Representing real-world entities (like books, users, products)
  • Creating configurations and settings
  • Building classes and modules
  • APIs and data manipulation

🔹 1. Shallow Cloning

Copies only the top-level properties. If a property is an object, it copies the reference, not the actual nested object.

✅ Methods for Shallow Clone:

a. Using Object.assign()

let original = { name: "Alice", age: 25 };
let clone = Object.assign({}, original);

b. Using Spread Operator (...)

let original = { name: "Alice", age: 25 };
let clone = { ...original };
⚠️ Limitation: Nested objects are not deeply copied.
let obj = { user: { name: "Bob" } };
let copy = { ...obj };
copy.user.name = "John";
console.log(obj.user.name); // John (affects original)

🔹 2. Deep Cloning

Creates an entirely independent copy, including all nested objects.

✅ Methods for Deep Clone:

a. Using JSON

let original = { name: "Alice", details: { age: 25 } };
let clone = JSON.parse(JSON.stringify(original));
⚠️ Limitations:
  • Cannot clone functions, undefined, Date, Map, Set, etc.
  • Loses prototype chain.

b. Using Structured Clone API (Modern)

let deepClone = structuredClone(originalObject);
✅ Supports most data types like Date, Map, Set, etc.

c. Manual Recursion (Custom Deep Clone Function)

function deepClone(obj) {
  if (obj === null || typeof obj !== "object") return obj;

  let clone = Array.isArray(obj) ? [] : {};

  for (let key in obj) {
    clone[key] = deepClone(obj[key]);
  }

  return clone;
}
