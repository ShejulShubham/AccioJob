🔹 PART 0: SQL & Database Fundamentals (Foundations)

0.1 What Problem Does a Database Solve?

Imagine this situation 👇

You’re building:

  • a website
  • a mobile app
  • an admin panel
  • a backend system

You need to store:

  • users
  • passwords
  • orders
  • payments
  • logs
  • attendance
  • scores
  • messages

❌ Bad way (real beginners do this)

  • Excel files
  • JSON files
  • Hardcoded data in code

❌ Problems

  • Data gets lost
  • Multiple users can’t access safely
  • Searching becomes slow
  • No security
  • No structure

✅ Correct way (industry way)

➡️ Use a Database

0.2 What is a Database?

Database = a structured place to store data permanently

Think of it like:

  • Excel, but super powerful
  • Handles millions of rows
  • Used by multiple users at the same time
  • Safe & fast

Real Examples

  • Instagram → users, posts, likes
  • Amazon → products, orders
  • Zomato → restaurants, orders
  • Your company → employees, attendance

0.3 What is a Table?

A table stores data in rows and columns.

Example: users table

idnameemailis_active
1Jayjay@mail.comtrue
2Nehaneha@mail.comtrue

Important terms

  • Column → name, email, is_active
  • Row → one complete user
  • Record → same as row

📌 Industry language:

“Insert a record”, “Fetch rows”, “Update user record”

0.4 What is SQL?

SQL = Structured Query Language

SQL is not a programming language like JS or Java.

SQL is:

  • a query language
  • used to talk to databases
  • used to:
    • insert data
    • read data
    • update data
    • delete data

Simple Example

SELECT * FROM users;

Meaning:

“Give me all data from users table”

0.5 SQL vs MySQL (VERY IMPORTANT)

Many beginners get confused here.

SQL

  • A language
  • Rules + syntax
  • Same everywhere (mostly)

MySQL

  • A database software
  • Understands SQL
  • Stores data physically on disk

So:

You write SQL,

MySQL executes it.

Real Analogy

  • English → Language
  • Human → understands English

➡️ SQL → Language

➡️ MySQL → understands SQL

0.6 What is MySQL and Why Industry Uses It?

MySQL is:

  • Open source
  • Fast
  • Reliable
  • Used in real companies

Where MySQL is used

  • Backend APIs (Node.js, Java, PHP)
  • Admin dashboards
  • Reporting tools
  • Production apps

Companies using MySQL

  • Startups
  • SaaS products
  • Internal tools
  • ERP systems

0.7 How Does Data Flow in Real Applications?

Let’s understand big picture (very important).

Step-by-step

  1. User opens website
  1. Clicks Login
  1. Frontend sends request
  1. Backend runs SQL query
  1. MySQL returns data
  1. Backend sends response to frontend

📌 SQL runs ONLY on backend, never in browser.

0.8 Does Entire App Download From Server?

This clears a BIG confusion.

What actually happens

  • Browser downloads:
    • index.html
    • JS files
    • CSS
  • Database data ❌ NOT downloaded

Data is fetched:

  • on demand
  • via API calls
  • using SQL at backend

So:

Database stays on server

SQL runs on server

0.9 Why SQL Is Still Mandatory in 2025

People say:

“ORM use karo, SQL nahi”

Reality:

  • ORM internally uses SQL
  • Debugging requires SQL
  • Performance tuning requires SQL
  • Interviews ask SQL

📌 Strong SQL = strong backend developer

0.10 Mental Model You Must Lock In

Before moving forward, lock this in your head:

Application → Backend → SQL → MySQL → Data

You:

  • write SQL
  • MySQL:
  • stores & retrieves data

🧪 Quick Check

  1. Is SQL a database?
  1. Is MySQL a language?

2.4 Common Datatypes (Only What Industry Uses Daily)

We will not overload you.

1️⃣ INT

Used for:

  • IDs
  • counts
  • numbers
id INT

2️⃣ VARCHAR(length)

Used for:

  • names
  • email
  • titles
name VARCHAR(100)

📌 Why length matters:

  • Saves memory
  • Improves performance

3️⃣ TEXT

Used for:

  • descriptions
  • comments
  • messages
bio TEXT

4️⃣ BOOLEAN

Used for:

  • flags
  • yes/no values
is_active BOOLEAN

Internally:

  • true → 1
  • false → 0

5️⃣ DATE

Used for:

  • birth date
  • joining date
joining_date DATE

6️⃣ DATETIME

Used for:

  • created_at
  • updated_at
  • logs
created_at DATETIME

7️⃣ DECIMAL

Used for:

  • salary
  • price
  • money
salary DECIMAL(10,2)

📌 NEVER use FLOAT for money.

2.5 Constraints (Rules That Protect Your Data)

Constraints = data safety rules

🔹 PRIMARY KEY (MOST IMPORTANT)

Rules:

  • Uniquely identifies a row
  • Cannot be NULL
  • Only ONE per table
id INT PRIMARY KEY

Industry meaning:

“This row can be uniquely referenced”

🔹 AUTO_INCREMENT

Used with IDs.

id INT AUTO_INCREMENT

Meaning:

MySQL automatically increases value

🔹 NOT NULL

Prevents missing data.

name VARCHAR(100) NOT NULL

🔹 UNIQUE

Prevents duplicates.

email VARCHAR(150) UNIQUE

🔹 DEFAULT

Sets default value.

is_active BOOLEAN DEFAULT true
