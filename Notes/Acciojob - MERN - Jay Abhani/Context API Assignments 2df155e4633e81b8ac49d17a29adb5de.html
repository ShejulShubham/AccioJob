✅ ASSIGNMENT 1 — Global Language Selector

🎯 Concepts Tested

  • createContext
  • Context.Provider
  • useContext
  • Avoiding prop drilling
  • Single global state

🧠 Problem Statement

You need to build a Global Language Selector using React Context API.

The application supports two languages:

  • English ("en")
  • Spanish ("es")

Once the language is changed:

  • Any component in the app should be able to access the selected language
  • No props should be passed manually between components

📂 Expected Folder Structure

Students must create the following structure:

src/
├── context/
│   ├── LanguageContext.js
│   └── LanguageProvider.js
│
├── components/
│   └── Greeting.jsx
│
└── App.jsx

📝 Detailed Instructions

  1. Create a LanguageContext using createContext.
  1. Create a LanguageProvider component:
    • Maintain a state called language
    • Default value should be "en"
    • Create a function toggleLanguage that switches between "en" and "es"
    • Pass both language and toggleLanguage through the Provider value.
  1. Wrap the entire application (App) with LanguageProvider.
  1. Inside Greeting component:
    • Use useContext to access the current language.
    • Display:
      • "Hello" when language is "en"
      • "Hola" when language is "es"
    • Add a button to change the language using toggleLanguage.
  1. Do not pass any props between components.
  1. The language state must be global and controlled only via context.

✅ Expected Output (Functional Behavior)

  • App loads with text: Hello
  • Clicking the button switches text to: Hola
  • Clicking again switches back to: Hello
  • No props are used anywhere in the app

✅ ASSIGNMENT 2 — Global Notification System

🎯 Concepts Tested

  • Part 2
  • Global state using Context
  • One context → multiple components
  • Conditional rendering based on global state
  • No prop drilling

🧠 Problem Statement

You need to build a Global Notification System using React Context API.

The application should have:

  • One component that can trigger a notification
  • Another component that can display and clear the notification
  • The notification state should be global

📂 Expected Folder Structure

Students must create the following structure:

src/
├── context/
│   ├── NotificationContext.js
│   └── NotificationProvider.js
│
├── components/
│   ├── NotificationControls.jsx
│   └── NotificationBar.jsx
│
└── App.jsx

📝 Detailed Instructions

  1. Create a NotificationContext using createContext.
  1. Create a NotificationProvider component:
    • Maintain a state called message
    • Initial value should be null
    • Create two functions:
      • showMessage(text) → sets the notification message
      • clearMessage() → clears the notification
    • Pass message, showMessage, and clearMessage through the Provider.
  1. Wrap the entire application (App) with NotificationProvider.
  1. In NotificationControls component:
    • Add a button that triggers a notification message using showMessage.
  1. In NotificationBar component:
    • Read the message from context.
    • If message is null, render nothing.
    • If a message exists, display it along with a button to clear it.
  1. Do not pass props between NotificationControls and NotificationBar.

✅ Expected Output (Functional Behavior)

  • App loads with no notification visible
  • Clicking the button shows a notification message
  • Clicking the close button removes the notification
  • Notification state is shared globally

✅ ASSIGNMENT 4 — Global Counter Across Multiple Components

🎯 Concepts Tested

  • Part 1
  • Single Context creation
  • Global state sharing
  • Multiple components consuming the same context
  • No prop drilling

🧠 Problem Statement

You need to build a Global Counter System using React Context API.

The application should have:

  • One global counter value
  • One component to display the counter
  • Another component to update the counter

Both components must always stay in sync using context, not props.

📂 Expected Folder Structure

Students must create the following structure:

src/
├── context/
│   ├── CounterContext.js
│   └── CounterProvider.js
│
├── components/
│   ├── CounterDisplay.jsx
│   └── CounterControls.jsx
│
└── App.jsx

📝 Detailed Instructions (For Students)

  1. Create a CounterContext using createContext.
  1. Create a CounterProvider component:
    • Maintain a state called count (number).
    • Initial value should be 0.
    • Create two functions:
      • increment() → increases count by 1
      • decrement() → decreases count by 1
    • Pass count, increment, and decrement via the Provider value.
  1. Wrap the entire application (App) with CounterProvider.
  1. In CounterDisplay component:
    • Use useContext to read and display the counter value.
  1. In CounterControls component:
    • Use useContext to access increment and decrement.
    • Add buttons to increase and decrease the counter.
  1. Do not pass props between CounterDisplay and CounterControls.

✅ Expected Output (Functional Behavior)

  • App loads showing counter value 0
  • Clicking Increase increments the counter
  • Clicking Decrease decrements the counter
  • Both components always show the same updated value
  • No props are used anywhere

✅ ASSIGNMENT 5 — Global Feature Toggle System

🎯 Concepts Tested

  • Global state using Context
  • Conditional rendering based on global state
  • One context → multiple consumers
  • No prop drilling

🧠 Problem Statement

You need to build a Global Feature Toggle System using React Context API.

The application has a global feature flag:

  • Advanced Mode (true / false)

This flag should:

  • Be toggled from one component
  • Be read from another component
  • Control what content is shown in the UI

📂 Expected Folder Structure

Students must create the following structure:

src/
├── context/
│   ├── FeatureContext.js
│   └── FeatureProvider.js
│
├── components/
│   ├── FeatureToggle.jsx
│   └── FeatureStatus.jsx
│
└── App.jsx

📝 Detailed Instructions (For Students)

  1. Create a FeatureContext using createContext.
  1. Create a FeatureProvider component:
    • Maintain a boolean state called isAdvanced.
    • Initial value should be false.
    • Create a function toggleFeature() to switch the value.
    • Pass isAdvanced and toggleFeature via the Provider value.
  1. Wrap the entire application (App) with FeatureProvider.
  1. In FeatureToggle component:
    • Use useContext to access toggleFeature.
    • Add a button to turn Advanced Mode ON / OFF.
  1. In FeatureStatus component:
    • Use useContext to read isAdvanced.
    • Conditionally render:
      • "Advanced Mode Enabled" when true
      • "Basic Mode Enabled" when false
  1. Do not pass any props between components.

✅ Expected Output (Functional Behavior)

  • App loads with Basic Mode Enabled
  • Clicking the toggle button switches to Advanced Mode Enabled
  • Clicking again switches back to Basic Mode Enabled
  • Feature state is shared globally
  • UI updates automatically based on context state

✅ Solution — Assignment 5

📁 context/FeatureContext.js

import { createContext } from "react";

export const FeatureContext = createContext();

📁 context/FeatureProvider.js

import { useState } from "react";
import { FeatureContext } from "./FeatureContext";

export default function FeatureProvider({ children }) {
  const [isAdvanced, setIsAdvanced] = useState(false);

  const toggleFeature = () => {
    setIsAdvanced((prev) => !prev);
  };

  return (
    <FeatureContext.Provider value={{ isAdvanced, toggleFeature }}>
      {children}
    </FeatureContext.Provider>
  );
}

📁 components/FeatureToggle.jsx

import { useContext } from "react";
import { FeatureContext } from "../context/FeatureContext";

export default function FeatureToggle() {
  const { toggleFeature } = useContext(FeatureContext);

  return (
    <button onClick={toggleFeature}>
      Toggle Advanced Mode
    </button>
  );
}

📁 components/FeatureStatus.jsx

import { useContext } from "react";
import { FeatureContext } from "../context/FeatureContext";

export default function FeatureStatus() {
  const { isAdvanced } = useContext(FeatureContext);

  return (
    <p>
      {isAdvanced
        ? "Advanced Mode Enabled"
        : "Basic Mode Enabled"}
    </p>
  );
}

📁 App.jsx

import FeatureProvider from "./context/FeatureProvider";
import FeatureToggle from "./components/FeatureToggle";
import FeatureStatus from "./components/FeatureStatus";

export default function App() {
  return (
    <FeatureProvider>
      <FeatureToggle />
      <FeatureStatus />
    </FeatureProvider>
  );
}

✅ ASSIGNMENT 6 — Dashboard Filters using Multiple Contexts

🎯 Concepts Tested

  • Multiple independent contexts
  • Separation of concerns
  • One component consuming more than one context
  • Clean global state design
  • No prop drilling

🧠 Problem Statement

You need to build a Dashboard Filter Panel using two separate global contexts.

The dashboard has two independent filters:

  1. Sort Order
    • "ascending" or "descending"
  1. View Mode
    • "compact" or "expanded"

Both filters must:

  • Be stored in separate contexts
  • Be available globally
  • Be used together inside the same component

📂 Expected Folder Structure

Students must create the following structure:

src/
├── context/
│   ├── SortContext.js
│   ├── SortProvider.js
│   ├── ViewContext.js
│   └── ViewProvider.js
│
├── components/
│   └── FilterPanel.jsx
│
└── App.jsx

📝 Detailed Instructions (For Students)

  1. Create SortContext:
    • Use createContext.
    • Maintain a state called sortOrder.
    • Initial value should be "ascending".
    • Create a function toggleSort() that switches between:
      • "ascending" ↔ "descending".
  1. Create ViewContext:
    • Use createContext.
    • Maintain a state called viewMode.
    • Initial value should be "compact".
    • Create a function toggleView() that switches between:
      • "compact" ↔ "expanded".
  1. Wrap the application with both providers.
  1. Inside FilterPanel component:
    • Use useContext to consume SortContext.
    • Use useContext to consume ViewContext.
    • Display the current sortOrder and viewMode.
    • Add buttons to toggle each filter independently.
  1. Do not merge both states into a single context.
  1. Do not pass any props between components.

✅ Expected Output (Functional Behavior)

  • App displays:
    • Current sort order
    • Current view mode
  • Clicking Toggle Sort switches:
    • ascending → descending → ascending
  • Clicking Toggle View switches:
    • compact → expanded → compact
  • Both filters work independently
  • One component consumes two contexts cleanly

✅ Solution — Assignment 6

📁 context/SortContext.js

import { createContext } from "react";

export const SortContext = createContext();

📁 context/SortProvider.js

import { useState } from "react";
import { SortContext } from "./SortContext";

export default function SortProvider({ children }) {
  const [sortOrder, setSortOrder] = useState("ascending");

  const toggleSort = () => {
    setSortOrder((prev) =>
      prev === "ascending" ? "descending" : "ascending"
    );
  };

  return (
    <SortContext.Provider value={{ sortOrder, toggleSort }}>
      {children}
    </SortContext.Provider>
  );
}

📁 context/ViewContext.js

import { createContext } from "react";

export const ViewContext = createContext();

📁 context/ViewProvider.js

import { useState } from "react";
import { ViewContext } from "./ViewContext";

export default function ViewProvider({ children }) {
  const [viewMode, setViewMode] = useState("compact");

  const toggleView = () => {
    setViewMode((prev) =>
      prev === "compact" ? "expanded" : "compact"
    );
  };

  return (
    <ViewContext.Provider value={{ viewMode, toggleView }}>
      {children}
    </ViewContext.Provider>
  );
}

📁 components/FilterPanel.jsx

import { useContext } from "react";
import { SortContext } from "../context/SortContext";
import { ViewContext } from "../context/ViewContext";

export default function FilterPanel() {
  const { sortOrder, toggleSort } = useContext(SortContext);
  const { viewMode, toggleView } = useContext(ViewContext);

  return (
    <>
      <p>Sort Order: {sortOrder}</p>
      <button onClick={toggleSort}>Toggle Sort</button>

      <p>View Mode: {viewMode}</p>
      <button onClick={toggleView}>Toggle View</button>
    </>
  );
}

📁 App.jsx

import SortProvider from "./context/SortProvider";
import ViewProvider from "./context/ViewProvider";
import FilterPanel from "./components/FilterPanel";

export default function App() {
  return (
    <SortProvider>
      <ViewProvider>
        <FilterPanel />
      </ViewProvider>
    </SortProvider>
  );
}
