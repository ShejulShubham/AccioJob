Searching Algos

🔍 Linear Search in JavaScript – Explained in Detail

✅ What is Linear Search?

Linear Search (also known as sequential search) is the simplest searching algorithm where you iterate through each element of an array to find a target value.

  • Time Complexity: O(n)
  • Space Complexity: O(1)
  • Best case: O(1) (element is at the first index)
  • Worst case: O(n) (element not found or at the end)

🧠 How It Works

  • Start from the first element.
  • Compare it with the target value.
  • If matched, return the index (or value).
  • If not, move to the next element.
  • If the end is reached and no match is found, return 1 or a "not found" message.

🧪 Basic Example – Return Index

function linearSearch(arr, target) {
  for (let i = 0; i < arr.length; i++) {
    // Compare current element with target
    if (arr[i] === target) {
      return i; // Return the index if found
    }
  }
  return -1; // Return -1 if not found
}

let numbers = [5, 3, 8, 4, 9];
console.log(linearSearch(numbers, 4)); // Output: 3
console.log(linearSearch(numbers, 7)); // Output: -1

🧪 Modified Version – Return Boolean

function isFound(arr, target) {
  for (let element of arr) {
    if (element === target) {
      return true;
    }
  }
  return false;
}

console.log(isFound([1, 2, 3, 4], 3)); // true
console.log(isFound([1, 2, 3, 4], 9)); // false

🧪 Linear Search on Strings

function linearSearchString(str, char) {
  for (let i = 0; i < str.length; i++) {
    if (str[i] === char) {
      return i;
    }
  }
  return -1;
}

console.log(linearSearchString("hello", "e")); // 1
console.log(linearSearchString("hello", "z")); // -1

🔁 Linear Search with Objects/Array of Objects

let users = [
  { id: 1, name: "Ujjwal" },
  { id: 2, name: "Ravi" },
  { id: 3, name: "Amit" }
];

function findUserByName(arr, name) {
  for (let i = 0; i < arr.length; i++) {
    if (arr[i].name === name) {
      return arr[i];
    }
  }
  return null;
}

console.log(findUserByName(users, "Ravi")); // { id: 2, name: "Ravi" }

📌 When to Use Linear Search?

  • When the array is small or unsorted
  • When no additional memory (like hash maps) should be used
  • When a simple solution is acceptable

❌ Limitations

  • Slow for large datasets
  • Inefficient compared to binary search (on sorted arrays)

✅ Summary

  • Linear Search is a simple and brute-force method.
  • It checks every element one by one until the target is found.
  • Use it for unsorted arrays or quick, small-scale lookups.
  • For sorted arrays, consider using binary search for better performance (O(log n)).

🔍 Binary Search in JavaScript – Explained in Detail

✅ What is Binary Search?

Binary Search is a highly efficient searching algorithm used to find an element in a sorted array. It works by repeatedly dividing the search range in half, eliminating half of the elements each time.

⚠️ Important: The array must be sorted in ascending or descending order.

🧠 How Binary Search Works

  1. Start with the low (start of the array) and high (end of the array) pointers.
  1. Find the middle element.
  1. If the middle value equals the target, return the index.
  1. If the target is less than the middle, search the left half.
  1. If the target is greater than the middle, search the right half.
  1. Repeat until low > high.

⏱️ Time & Space Complexity

ScenarioTime ComplexitySpace Complexity
Best CaseO(1)O(1)
Average CaseO(log n)O(1)
Worst CaseO(log n)O(1)

🛠️ Binary Search – Iterative Version

function binarySearch(arr, target) {
  let low = 0;
  let high = arr.length - 1;

  while (low <= high) {
    let mid = Math.floor((low + high) / 2);

    if (arr[mid] === target) {
      return mid; // Target found
    } else if (arr[mid] < target) {
      low = mid + 1; // Search right half
    } else {
      high = mid - 1; // Search left half
    }
  }

  return -1; // Target not found
}

let numbers = [2, 4, 6, 8, 10, 12, 14];
console.log(binarySearch(numbers, 10)); // Output: 4

🧠 Dry Run Example

Array: [2, 4, 6, 8, 10, 12, 14]

Target: 10

  • Iteration 1: low = 0, high = 6 → mid = 3 → arr[3] = 8

    → 10 > 8 → search in right half (low = 4)

  • Iteration 2: low = 4, high = 6 → mid = 5 → arr[5] = 12

    → 10 < 12 → search in left half (high = 4)

  • Iteration 3: low = 4, high = 4 → mid = 4 → arr[4] = 10 → found

🧑‍💻 Binary Search – Recursive Version

function binarySearchRecursive(arr, target, low = 0, high = arr.length - 1) {
  if (low > high) return -1;

  let mid = Math.floor((low + high) / 2);

  if (arr[mid] === target) return mid;
  else if (arr[mid] > target) {
    return binarySearchRecursive(arr, target, low, mid - 1);
  } else {
    return binarySearchRecursive(arr, target, mid + 1, high);
  }
}

console.log(binarySearchRecursive([1, 3, 5, 7, 9], 5)); // Output: 2

🧪 Handling Edge Cases

  • Array is empty → return -1
  • Target not found → return -1
  • Array not sorted → binary search won't work

⚠️ Sorted Input Required

Always ensure the array is sorted before applying binary search.

let arr = [10, 3, 7, 1];
arr.sort((a, b) => a - b); // Sort before binary search

✅ Use Cases

  • Searching in large sorted datasets
  • Searching in dictionaries or sorted user lists
  • Implementing efficient lookup features (e.g., auto-suggestions)

❌ Limitations

  • Only works on sorted arrays
  • Less efficient for small arrays (linear search may be faster)
  • Not useful if data changes frequently (requires re-sorting)

📌 Summary

  • Binary Search is fast (O(log n)) for sorted data.
  • It repeatedly halves the search space.
  • Prefer iterative for simplicity, recursive for conceptual clarity.
  • Always ensure the array is sorted before applying it.

🔍 What is a Map?

A Map is a collection of elements where each element is stored as a key-value pair. It remembers the original insertion order of the keys, meaning you can iterate over the map in the order in which elements were added.

let map = new Map();

✅ Why Use Map Instead of Object?

FeatureMapObject
Key TypesAny (objects, functions, etc)Only strings or symbols
Order of KeysOrderedUnordered
Size propertymap.sizeNeed manual tracking
Iterationmap.forEach, for...ofMore cumbersome
Performance (frequent insert/delete)BetterSlower in some cases

🛠️ Creating a Map

1. Empty Map

let myMap = new Map();

2. Map with initial values

let myMap = new Map([
  ['name', 'Ujjwal'],
  ['age', 25],
  ['isStudent', true]
]);

📌 Key Methods of Map

1. set(key, value) → Adds or updates an entry

myMap.set('language', 'JavaScript');
myMap.set(10, 'ten'); // Key can be a number
myMap.set(true, 'yes'); // Boolean key

2. get(key) → Retrieves the value

console.log(myMap.get('language')); // JavaScript

3. has(key) → Checks if a key exists

console.log(myMap.has(10)); // true

4. delete(key) → Removes a specific element

myMap.delete(10);

5. clear() → Removes all elements

myMap.clear();

6. size → Returns the number of elements

console.log(myMap.size); // 0

🔁 Iterating a Map

let map = new Map([
  ['a', 1],
  ['b', 2],
  ['c', 3]
]);

// 1. for...of (entries)
for (let [key, value] of map) {
  console.log(`${key} => ${value}`);
}

// 2. keys only
for (let key of map.keys()) {
  console.log(key);
}

// 3. values only
for (let value of map.values()) {
  console.log(value);
}

// 4. forEach
map.forEach((value, key) => {
  console.log(`${key}: ${value}`);
});

💡 Key Features

  1. Preserves Insertion Order:
    let m = new Map();
m.set('x', 1);
m.set('y', 2);
// Iteration will return x before y
  1. Accepts Objects as Keys:
    let obj = { id: 1 };
m.set(obj, 'Object Value');
console.log(m.get(obj)); // Object Value
  1. Distinct NaN keys allowed:
    m.set(NaN, 'not a number');
console.log(m.get(NaN)); // not a number

🔄 Map vs Object - When to Use What?

Use CaseChoose
Frequent addition/removal of entriesMap
Keys need to be non-string valuesMap
JSON serialization neededObject
Simple key-value pairs with string keysObject

✅ Practical Example

let userRoles = new Map();

userRoles.set('Ujjwal', 'Admin');
userRoles.set('Ravi', 'Editor');
userRoles.set('Amit', 'Viewer');

// Check Role
if (userRoles.get('Ravi') === 'Editor') {
  console.log('Ravi can edit content.');
}

🧠 Summary

  • Map is ideal for more complex key-value pairs.
  • It maintains insertion order, unlike plain objects.
  • It provides better performance for large-scale operations or frequent additions/removals.
  • It supports any data type as a key, including functions and objects.

Problem 1: Count Frequency of Elements in an Array

Description:

Given an array of numbers, count how many times each number appears.

Input: [1, 2, 2, 3, 1, 4, 2]

Output: {1 => 2, 2 => 3, 3 => 1, 4 => 1}

function countFrequency(arr) {
  let map = new Map();

  for (let num of arr) {
    // If number exists, increment count, else set to 1
    map.set(num, (map.get(num) || 0) + 1);
  }

  return map;
}

console.log(countFrequency([1, 2, 2, 3, 1, 4, 2]));

Problem 2: Find First Non-Repeating Element

Description:

Return the first element in the array that appears only once.

Input: [2, 3, 4, 2, 3, 5, 4]

Output: 5

function firstNonRepeating(arr) {
  let freqMap = new Map();

  for (let num of arr) {
    freqMap.set(num, (freqMap.get(num) || 0) + 1);
  }

  for (let num of arr) {
    if (freqMap.get(num) === 1) return num;
  }

  return null;
}

console.log(firstNonRepeating([2, 3, 4, 2, 3, 5, 4]));

Problem 3: Check if Two Strings are Anagrams

Description:

Two strings are anagrams if they contain the same characters with the same frequency.

Input: "listen", "silent"

Output: true

function areAnagrams(str1, str2) {
  if (str1.length !== str2.length) return false;

  let map = new Map();

  for (let char of str1) {
    map.set(char, (map.get(char) || 0) + 1);
  }

  for (let char of str2) {
    if (!map.has(char)) return false;
    map.set(char, map.get(char) - 1);
    if (map.get(char) < 0) return false;
  }

  return true;
}

console.log(areAnagrams("listen", "silent"));

Problem 5: Remove Duplicates from an Array

Description:

Use a Map to filter unique elements and preserve the order.

Input: [1, 2, 2, 3, 1, 4]

Output: [1, 2, 3, 4]

function removeDuplicates(arr) {
  let map = new Map();

  for (let num of arr) {
    map.set(num, true); // Only one key per number
  }

  return Array.from(map.keys());
}

console.log(removeDuplicates([1, 2, 2, 3, 1, 4]));
