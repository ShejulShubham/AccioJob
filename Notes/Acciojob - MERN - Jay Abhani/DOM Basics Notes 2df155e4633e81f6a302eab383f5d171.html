DOM Basics Notes

Intro to DOM and Selectors

Part 1: What is the DOM?

Definition:

The DOM (Document Object Model) is a tree-like structure that the browser creates from your HTML code, allowing JavaScript to interact with elements on the page.

Analogy:

  • Family Tree: HTML tags are like family members in a family tree. Each tag is a node, and they can have parents, children, and siblings.
  • House Structure: HTML is like a house blueprint. Each element (room, window, door) can be accessed and modified.

Visual Demo:

Show a simple HTML file and its corresponding DOM tree diagram.

<!-- Sample HTML -->
<div id="main">
  <h1 class="title">Welcome</h1>
  <p>This is a paragraph.</p>
  <button>Click me</button>
</div>

Part 2: How the Browser Creates the DOM

  1. HTML is loaded.
  1. The browser parses it top to bottom.
  1. Creates nodes (elements, text, comments).
  1. Builds a tree-like model (DOM tree).

Show Chrome DevTools (Elements tab) to help students see the live DOM structure.

Part 3: What Are Nodes and Elements

  • Node: Any item in the DOM tree (element, text, comment).
  • Element Node: A tag like <div>, <h1>.
  • Text Node: The actual text inside an element.
  • Visual Tip: Use a DOM inspector (DevTools) to show them.

Part 4: Selecting Elements in the DOM

Use the same HTML structure for all examples for consistency.

<div id="main">
  <h1 class="title">Welcome</h1>
  <p>This is a paragraph.</p>
  <button>Click me</button>
</div>

1. getElementById()

const mainDiv = document.getElementById("main");
console.log(mainDiv);

Visual: Highlight the entire <div> block.

2. getElementsByClassName()

const titles = document.getElementsByClassName("title");
console.log(titles[0]);

Visual: Point out that it returns a collection (even if there's only one element).

3. getElementsByTagName()

const paragraphs = document.getElementsByTagName("p");
console.log(paragraphs[0]);

Visual: Show how it selects all <p> elements.

4. querySelector()

const title = document.querySelector(".title");
console.log(title);

Visual: Explain it's a CSS-style selector and returns only the first match.

5. querySelectorAll()

const allTitles = document.querySelectorAll(".title");
console.log(allTitles);

Visual: Highlight that it returns a NodeList (you can loop through it).

👨‍🏫 Lesson: Creating and Modifying the DOM

🔧 DOM Method 1: createElement()

✅ What it does:

Creates a new HTML element node in memory — but it's not visible on the page yet.

🧠 Analogy:

Like crafting a new LEGO block 🧱 but not placing it on the table (page) yet.

📄 Example:

const newPara = document.createElement("p"); // Creates <p></p>

🔧 DOM Method 2: appendChild()

✅ What it does:

Adds the new element to an existing element on the page (DOM tree).

🧠 Analogy:

You're taking the LEGO block 🧱 and attaching it to the house 🏠.

📄 Example:

newPara.textContent = "This is a new paragraph.";
document.body.appendChild(newPara);
Now the new paragraph is visible on the webpage under <body>!

🔧 DOM Method 3: removeChild()

✅ What it does:

Removes a child element from its parent.

🧠 Analogy:

You're detaching a LEGO block from the house 🏠 and putting it away.

📄 Example:

const button = document.querySelector("button");
document.body.removeChild(button);

🧪👩‍💻 Interactive Classroom Activity

🧱 HTML Setup:

<div id="container">
  <h2>My DOM Playground</h2>
  <button id="add">Add Paragraph</button>
  <button id="remove">Remove Paragraph</button>
</div>

📜 JavaScript:

const container = document.getElementById("container");
const addBtn = document.getElementById("add");
const removeBtn = document.getElementById("remove");
let newPara = null;

addBtn.addEventListener("click", () => {
  newPara = document.createElement("p");
  newPara.textContent = "This is a new paragraph added!";
  container.appendChild(newPara); // Adds it to DOM
});

removeBtn.addEventListener("click", () => {
  if (newPara) {
    container.removeChild(newPara); // Removes from DOM
    newPara = null;
  }
});




<script>
  const container = document.getElementById("container");
  const addBtn = document.getElementById("add");
  const removeBtn = document.getElementById("remove");

  const addedParas = []; // Array to store all added paragraphs

  addBtn.addEventListener("click", () => {
    const para = document.createElement("p");
    para.textContent = "Acciojob";
    container.appendChild(para);
    addedParas.push(para); // Store reference of added paragraph
  });

  removeBtn.addEventListener("click", () => {
    const lastPara = addedParas.pop(); // Get last added paragraph
    if (lastPara) {
      container.removeChild(lastPara);
    }
  });
</script>

👨‍🏫 Lesson: Adding Text and HTML Content to the DOM

🧱 There are Two Common Ways:

MethodWhat It DoesUse Case
textContentAdds plain textUse when adding user input or content with no HTML
innerHTMLAdds HTML contentUse when you need formatting (e.g., bold, italic, spans)

🔧 1. textContent

✅ What it does:

  • Sets or updates the plain text inside an element.
  • Treats everything as text — does not parse HTML tags.

🧠 Analogy:

🗣️ Speaking words directly — no formatting.

📄 Example:

const heading = document.createElement("h2");
heading.textContent = "<i>Hello</i> Students!"; // Shows the raw tags
document.body.appendChild(heading);

🧾 Output:

<i>Hello</i> Students! (Just plain text)

🔧 2. innerHTML

✅ What it does:

  • Sets or updates the HTML structure inside an element.
  • Parses and renders tags.

🧠 Analogy:

📝 Writing rich text with styles like bold or italics.

📄 Example:

const subHeading = document.createElement("h3");
subHeading.innerHTML = "<i>Welcome</i> to <b>JavaScript DOM</b>";
document.body.appendChild(subHeading);

🧾 Output:

Welcome to JavaScript DOM

❗ Important Differences:

FeaturetextContentinnerHTML
Treats tags as text✅ Yes❌ No
Parses HTML❌ No✅ Yes
Safer for user input✅ Yes❌ No (can be unsafe if from users)

🧪👨‍💻 Interactive Activity: Text vs HTML

🧱 HTML:

<button id="add-text">Add Plain Text</button>
<button id="add-html">Add HTML Content</button>
<div id="output"></div>

📜 JavaScript:

const textBtn = document.getElementById("add-text");
const htmlBtn = document.getElementById("add-html");
const outputDiv = document.getElementById("output");

textBtn.addEventListener("click", () => {
  outputDiv.textContent = "<strong>This is plain text</strong>";
});

htmlBtn.addEventListener("click", () => {
  outputDiv.innerHTML = "<strong>This is HTML content</strong>";
});

🧠 What Students Observe:

  • First button prints HTML tags literally.
  • Second button renders bold text.

✅ Final Tip:

Use textContent when:

  • You want just text, no formatting.
  • You're working with user-generated input (to avoid XSS attacks).

Use innerHTML when:

  • You want to insert HTML tags for styling or structure.

👨‍🏫 Lesson: Editing Attributes in the DOM

✅ Why Attributes Matter:

Attributes like id, class, href, src, and type help define element behavior and appearance. JavaScript can dynamically change these attributes using special DOM methods.

🔧 DOM Method 1: setAttribute()

✅ What it does:

Sets or updates the value of an attribute on an element.

🧠 Analogy:

📌 Like sticking a label or changing the nameplate on an object.

📄 Example:

const btn = document.querySelector("button");
btn.setAttribute("id", "main-button"); // Adds or updates the 'id'

🔧 DOM Method 2: getAttribute()

✅ What it does:

Retrieves the current value of an attribute.

🧠 Analogy:

👓 Like reading the label on an object.

📄 Example:

const idValue = btn.getAttribute("id");
console.log(idValue); // "main-button"

🔧 DOM Method 3: removeAttribute()

✅ What it does:

Removes an existing attribute from an element.

🧠 Analogy:

❌ Like peeling off a sticker or removing a nameplate.

📄 Example:

btn.removeAttribute("id"); // Now the button has no 'id'

🧪👩‍💻 Interactive Activity: Play with Attributes

🧱 HTML Setup:

<button id="actionBtn" type="button">Click Me</button>
<div id="output"></div>

//set attribute
//get attribute
//remove attribute

📜 JavaScript:

const btn = document.getElementById("actionBtn");
const output = document.getElementById("output");

// 1. Set attribute
btn.addEventListener("click", () => {
  btn.setAttribute("class", "btn-primary");
  output.textContent = "Attribute 'class' set to 'btn-primary'";
});

// 2. Get attribute
btn.addEventListener("dblclick", () => {
  const typeValue = btn.getAttribute("type");
  output.textContent = `The 'type' of button is: ${typeValue}`;
});

// 3. Remove attribute (on right click)
btn.addEventListener("contextmenu", (e) => {
  e.preventDefault();
  btn.removeAttribute("class");
  output.textContent = "Attribute 'class' removed";
});

🧠 What Students Learn:

MethodActionReal-Life Analogy
setAttribute()Adds or updates a labelAdding a name tag
getAttribute()Reads the labelChecking a tag
removeAttribute()Removes the labelPeeling a sticker

✅ Bonus Tip:

You can also access some attributes directly:

btn.id = "main-button"; // Same as setAttribute
console.log(btn.id);    // Same as getAttribute

But using the methods is more flexible and universal, especially for custom attributes.

👨‍🏫 Lesson: Working with Classes – className vs classList

📦 Why Classes Matter in DOM

  • Classes are used to style elements (via CSS).
  • JavaScript allows you to add, remove, check, or toggle classes dynamically.

🔧 1. className

✅ What it does:

Gets or sets the entire class string of an element.

🧠 Analogy:

📋 Like writing a full list of tags on a label all at once.

📄 Example:

const box = document.getElementById("myBox");

console.log(box.className); // Outputs: "box blue shadow"

box.className = "box red"; // Replaces all classes with just "box red"

❗ Warning:

⚠️ Overwrites all existing classes. Not safe if you only want to change one class.

🔧 2. classList

✅ What it does:

Gives access to a collection of classes and provides helpful methods to manipulate them:

  • .add()
  • .remove()
  • .toggle()
  • .contains()
  • .replace()

🧠 Analogy:

🎮 Like using buttons to add/remove individual tags on an object.

📄 Example:

const box = document.getElementById("myBox");

box.classList.add("highlight");        // Adds a class
box.classList.remove("blue");          // Removes a class
box.classList.toggle("visible");       // Toggles a class on/off
console.log(box.classList.contains("shadow")); // true or false
box.classList.replace("red", "green"); // Replaces a class

🧪👨‍💻 Interactive Activity: Class Controller

🧱 HTML Setup:

<div id="myBox" class="box blue shadow">I'm a box</div>

<button id="add">Add "highlight"</button>
<button id="remove">Remove "blue"</button>
<button id="toggle">Toggle "rounded"</button>
<button id="check">Check "shadow"</button>

📜 JavaScript:

const box = document.getElementById("myBox");
const output = document.createElement("p");
document.body.appendChild(output);

document.getElementById("add").addEventListener("click", () => {
  box.classList.add("highlight");
  output.textContent = 'Class "highlight" added!';
});

document.getElementById("remove").addEventListener("click", () => {
  box.classList.remove("blue");
  output.textContent = 'Class "blue" removed!';
});

document.getElementById("toggle").addEventListener("click", () => {
  box.classList.toggle("rounded");
  output.textContent = 'Class "rounded" toggled!';
});

document.getElementById("check").addEventListener("click", () => {
  const hasShadow = box.classList.contains("shadow");
  output.textContent = hasShadow
    ? 'Yes! "shadow" class exists.'
    : '"shadow" class is missing.';
});

FeatureclassNameclassList
TypeStringDOMTokenList (array-like)

🧠 Final Tip:

  • Use classList when you want to manipulate individual classes dynamically.
  • Use className if you want to completely reset all classes.
