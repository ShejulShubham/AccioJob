🧪 Assignment 1: Basic GET and Render to UI

1. Problem

Fetch a list of posts from

https://jsonplaceholder.typicode.com/posts

and display the first 5 posts on the page as a list with title and body.

2. Requirements

  • Show "Loading..." while data is being fetched.
  • If fetch fails (offline / network), show: "Failed to load posts."
  • Don’t just console.log, actually update the DOM.

3. Solution

<div id="status"></div>
<ul id="posts"></ul>

<script>
const statusBox = document.getElementById("status");
const postsList = document.getElementById("posts");

statusBox.textContent = "Loading...";

fetch("https://jsonplaceholder.typicode.com/posts")
  .then(response => {
    // Handle server error (404, 500, etc.)
    if (!response.ok) {
      throw new Error("Server error: " + response.status);
    }
    return response.json();
  })
  .then(data => {
    statusBox.textContent = ""; // clear loading state

    const firstFive = data.slice(0, 5);

    // If array is empty
    if (firstFive.length === 0) {
      statusBox.textContent = "No posts found.";
      return;
    }

    firstFive.forEach(post => {
      const li = document.createElement("li");
      li.innerHTML = `<strong>${post.title}</strong><br>${post.body}`;
      postsList.appendChild(li);
    });
  })
  .catch(error => {
    console.error(error);
    statusBox.textContent = "Failed to load posts.";
  });
</script>

What you practiced:

  • fetch()
  • response.ok
  • .json()
  • loading + error UI

🧪 Assignment 3: Create (POST) a New Post

1. Problem

Create a simple form:

  • Title (text input)
  • Body (textarea)
  • Submit button

When user submits, send a POST request to

https://jsonplaceholder.typicode.com/posts

You should send { title, body, userId: 1 } as JSON.

Then show the response from the server on the page.

2. Requirements

  • Use method: "POST".
  • Send headers: Content-Type: "application/json".
  • Use JSON.stringify(...).
  • If server responds with success, show: "Post created with id: <id>".
  • If server responds with error status, show: "Server error: <status>".
  • If network fails, show: "Network error".

3. Solution

<h2>Create Post</h2>
<input id="postTitle" placeholder="Title"><br><br>
<textarea id="postBody" placeholder="Body"></textarea><br><br>
<button id="createBtn">Create Post</button>

<div id="postStatus"></div>

<script>
    const btnCreate = document.getElementById("createBtn");
    const statusBox = document.getElementById("postStatus");

    btnCreate.addEventListener("click", () => {
        const titleVal = document.getElementById("postTitle").value.trim();
        const bodyVal = document.getElementById("postBody").value.trim();

        if (!titleVal || !bodyVal) {
            statusBox.textContent = "Please fill all fields.";
            return;
        }

        statusBox.textContent = "Sending post...";

        fetch("https://jsonplaceholder.typicode.com/posts", {
            method: "POST",
            headers: {
                "Content-Type": "application/json" // telling server we send JSON
            },
            body: JSON.stringify({
                title: titleVal,
                body: bodyVal,
                userId: 1
            })
        })
            .then(res => {
                if (!res.ok) {
                    throw new Error("Server error: " + res.status);
                }
                return res.json();
            })
            .then(data => {
                statusBox.textContent = `Post created with id: ${data.id}`;
            })
            .catch(err => {
                console.error(err);
                if (err.message.startsWith("Server error")) {
                    statusBox.textContent = err.message;
                } else {
                    statusBox.textContent = "Network error";
                }
            });
    });
</script>

What you practiced:

  • Sending data to server → POST
  • Request headers
  • Request body
  • Handling success and failures differently

🧪 Assignment 4: Update (PUT) an Existing Post

1. Problem

You will "edit" a post with id 1 on JSONPlaceholder.

Create inputs:

  • New title
  • New body
  • A button: “Update Post 1”

When clicked:

  • Send a PUT request to

    https://jsonplaceholder.typicode.com/posts/1

  • Body should be a full updated object:
    {
  id: 1,
  title: "NEW_TITLE",
  body: "NEW_BODY",
  userId: 1
}

Show the updated result in the UI.

2. Requirements

  • Use method: "PUT"
  • Include headers, Content-Type: "application/json"
  • Handle server error codes and network errors like before.

3. Solution

<h2>Edit Post #1</h2>
<input id="editTitle" placeholder="New title"><br><br>
<textarea id="editBody" placeholder="New body"></textarea><br><br>
<button id="updateBtn">Update Post 1</button>

<div id="updateStatus"></div>

<script>
    const updateBtn = document.getElementById("updateBtn");
    const updateStatus = document.getElementById("updateStatus");

    updateBtn.addEventListener("click", () => {
        const newTitle = document.getElementById("editTitle").value.trim();
        const newBody = document.getElementById("editBody").value.trim();

        if (!newTitle || !newBody) {
            updateStatus.textContent = "Please fill both fields.";
            return;
        }

        updateStatus.textContent = "Updating post...";

        fetch("https://jsonplaceholder.typicode.com/posts/1", {
            method: "PUT",
            headers: {
                "Content-Type": "application/json"
            },
            body: JSON.stringify({
                id: 1,
                title: newTitle,
                body: newBody,
                userId: 1
            })
        })
            .then(res => {
                if (!res.ok) {
                    throw new Error("Server error: " + res.status);
                }
                return res.json();
            })
            .then(data => {
                updateStatus.textContent =
                    `Updated!\nTitle: ${data.title}\nBody: ${data.body}`;
            })
            .catch(err => {
                console.error(err);
                if (err.message.startsWith("Server error")) {
                    updateStatus.textContent = err.message;
                } else {
                    updateStatus.textContent = "Network error.";
                }
            });
    });
</script>

What you practiced:

  • PUT request
  • Full object update pattern (id, title, body, userId)
  • Reusing same error handling structure

🧪 Assignment 6: Show Loading State (Spinner / Message) Until Data Arrives

1. Problem

You’re building a “Random Quote” widget using this API:

https://api.quotable.io/random

Show:

  • "Loading quote..." while fetch() is in progress
  • When data is received, show the quote and author
  • If there’s any error (server or network), show "Could not load quote."

2. Requirements

  • Use DOM updates (no console-only solutions)
  • Use response.ok check

3. Solution

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <div id="quoteDiv"></div>

    <script>
        let quoteDiv = document.getElementById("quoteDiv");

        fetch("https://api.quotable.io/random")
        .then((res) => {
            if(!res.ok) {
                throw new Error("server error");
            }
            return res.json();
        })
        .then((data) => {
            quoteDiv.innerHTML = `<p>${data.content}</p><br>-<strong>${data.author}</strong>`;
        })
        .catch((err) => {
            quoteDiv.textContent = "Could not load quote.";
        })
    </script>
</body>
</html>

What you practiced:

  • Loading state before data
  • Clearing loading after success
  • Fallback message on error

This is exactly what you do in dashboards / widgets / cards.
