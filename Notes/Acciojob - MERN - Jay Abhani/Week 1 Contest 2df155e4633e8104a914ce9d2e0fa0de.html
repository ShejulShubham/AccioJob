Problem: Convert Days to Years, Months, and Days

Problem Statement

Write a program that takes a number of days as input and converts it into years, months, and remaining days. Assume:

1 year = 365 days1 month = 30 days

  • 1 year = 365 days
  • 1 month = 30 days

Input

A single integer d (1 ≤ d ≤ 10000) representing the number of days.

Output

Print the result in the format:

X years, Y months, Z days

Examples

Example 1: Input:

400

Output:

1 years, 1 months, 5 days

Example 2: Input:

800

Output:

2 years, 2 months, 10 days

const readline = require('readline').createInterface({
  input: process.stdin,
  output: process.stdout
});

readline.question('', input => {
  const d = Number(input);

  const years = Math.floor(d / 365);
  const remainingDaysAfterYears = d % 365;

  const months = Math.floor(remainingDaysAfterYears / 30);
  const days = remainingDaysAfterYears % 30;

  console.log(`${years} years, ${months} months, ${days} days`);

  readline.close();
});

✅ Solution:

const readline = require('readline').createInterface({
  input: process.stdin,
  output: process.stdout
});

let inputs = [];

readline.question('', d => {
  inputs.push(Number(d.trim()));
  readline.question('', m => {
    inputs.push(Number(m.trim()));
    readline.question('', y => {
      inputs.push(Number(y.trim()));
      readline.close();

      const day = inputs[0];
      const month = inputs[1];
      const year = inputs[2];

      // ✅ Basic checks
      if (month < 1 || month > 12 || day < 1 || day > 31 || year < 1 || year > 100000) {
        console.log("Invalid");
        return;
      }

      // ✅ Leap year check
      let isLeap = false;
      if ((year % 4 === 0 && year % 100 !== 0) || year % 400 === 0) {
        isLeap = true;
      }

      // ✅ Max days per month
      let maxDays;
      if (month === 2) {
        maxDays = isLeap ? 29 : 28;
      } else if (
        month === 4 || month === 6 ||
        month === 9 || month === 11
      ) {
        maxDays = 30;
      } else {
        maxDays = 31;
      }

      // ✅ Final check
      if (day <= maxDays) {
        console.log("Valid");
      } else {
        console.log("Invalid");
      }
    });
  });
});

Problem - 3

E-commerce company "ShopEasy" has an online shopping platform that allows users to purchase products from various categories. The company uses JavaScript to implement business logic on the client-side. One of the features of the platform is to offer discounts to customers based on their purchase history. The discounts are calculated using a complex expression involving multiple conditions and operators.

You are tasked with debugging an issue in the discount calculation logic, where the expression const result = 5 + 10 * 2 > 28 && (true || false) is not producing the expected output. Your goal is to evaluate this expression and explain step-by-step how you would arrive at the final result.

Assume that you have access to a JavaScript console where you can write code and execute it. You need to provide a clear explanation of how you would approach this problem, including any intermediate calculations or evaluations.

Please provide your approach to solve this problem.

Solution

the expression const result = 5 + 10 * 2 > 28 && (true || false) is not producing the correct result

Let’s understand what the business logic likely intended, then rewrite the expression so it behaves correctly.

🧠 Current Behavior:

const result = 5 + 10 * 2 > 28 && (true || false);

Evaluates to:

5 + 10 * 2 = 25
25 > 28 = false
false && (true || false) = false

💡 Likely Intention of Business Logic

If it’s a discount logic, the developer probably meant:

"If (5 + 10) * 2 > 28, and (true OR false), then grant discount"

But due to operator precedence, it’s not grouped correctly.

✅ Correct Expression with Parentheses:

const result = (5 + 10) * 2 > 28 && (true || false);

🔍 Evaluation:

(5 + 10) * 2 = 15 * 2 = 30
30 > 28 = true
(true || false) = true
true && true = ✅ true

✅ Final Answer:

const result = (5 + 10) * 2 > 28 && (true || false);

✔️ This will now produce true, which is likely the intended outcome.

Problem - 4

A social media company, "ConnectMe", wants to develop a feature to recommend the most popular post of the day based on the number of likes, comments, and shares. The feature should be able to take three distinct numbers as input, representing the number of likes, comments, and shares respectively, and return the greatest of the three numbers. The company wants to implement this feature using both ternary operators and switch-case statements in JavaScript.

Write a JavaScript code snippet that takes three distinct numbers as input and returns the greatest of the three numbers using both ternary operators and switch-case statements.

Assume that the input numbers are stored in variables likes, comments, and shares.

You can use the following markdown for writing your code:

// your code here

✅ Using Ternary Operator:

// Input values
const likes = 120;
const comments = 350;
const shares = 200;

// Using nested ternary to find the greatest
const greatest = (likes > comments)
  ? (likes > shares ? likes : shares)
  : (comments > shares ? comments : shares);

console.log("Greatest using ternary:", greatest);

✅ Using Switch-Case Statement:

// Input values
const likes = 120;
const comments = 350;
const shares = 200;

// First, find max value using Math.max (to simplify switch)
const max = Math.max(likes, comments, shares);

switch (max) {
  case likes:
    console.log("Greatest using switch: Likes =", likes);
    break;
  case comments:
    console.log("Greatest using switch: Comments =", comments);
    break;
  case shares:
    console.log("Greatest using switch: Shares =", shares);
    break;
  default:
    console.log("Something went wrong.");
}
