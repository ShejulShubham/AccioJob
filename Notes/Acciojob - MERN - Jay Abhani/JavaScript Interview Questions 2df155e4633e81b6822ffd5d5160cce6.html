1️⃣ (Theory) Explain Hoisting with an example.

👉 What will the following code print and why?

console.log(a);
var a = 10;

✅ Output: undefined

🧠 Explanation: Variable a is hoisted (i.e., declared at the top internally) but not initialized until the assignment line executes. Hence, its value is undefined at the time of the log.

2️⃣ (Predict Output) Understand Block Scope with let

👉 Predict the output of this program:

function test() {
  let x = 10;
  if (true) {
    let x = 20;
    console.log(x);
  }
  console.log(x);
}
test();

✅ Output:

20
10

🧠 Explanation: let creates a new variable each time for a different block. The inner x does not affect the outer x.

3️⃣ (Theory) Differentiate between == and ===.

👉 Explain with one example each where they give different results.

✅ Answer:

  • == checks for value equality after type conversion.
  • === checks for value and type equality (without conversion).
0 == "0"   // true
0 === "0"  // false

5️⃣ (Coding) Reverse each word in a sentence but keep the word order same.

Example:

Input: "Hello World"

Output: "olleH dlroW"

✅ Solution:

function reverseWords(str) {
  return str
    .split(" ")
    .map(word => word.split("").reverse().join(""))
    .join(" ");
}
console.log(reverseWords("Hello World"));

6️⃣ (Logic Puzzle) Async Behavior with setTimeout

  setTimeout(() => console.log(i), 1000);
}

🧠 Explanation: var is function-scoped. By the time callbacks execute, the loop has finished and i = 4.

Use let i to capture values per iteration.

7️⃣ (Theory) Explain Event Bubbling and Capturing in the DOM.

✅ Answer:

  • Capturing phase: Event travels from root → target.
  • Bubbling phase: Event travels back target → root.

    By default, addEventListener("click", fn, false) listens in the bubbling phase.

1️⃣ (Theory) Explain the difference between Function Declaration and Function Expression.

👉 Which one can you call before its definition? Why?

✅ Answer:

  • Function Declaration is fully hoisted — you can call it before it appears in the code.
  • Function Expression is assigned to a variable; only the variable name is hoisted (as undefined).
sayHello(); // ✅ Works
function sayHello() { console.log("Hi!"); }

sayHi(); // ❌ TypeError
var sayHi = function() { console.log("Hello!"); };

2️⃣ (Theory) What is an Execution Context in JavaScript? What does it contain?

✅ Answer:

Every time a function is called, JavaScript creates a new execution context with three parts:

  1. Variable Environment (variables & functions declared inside)
  1. Scope Chain (link to outer contexts)
  1. this Binding (the value of this for that call)

The context is pushed onto the call stack when the function starts and popped off when it finishes.

3️⃣ (Predict Output) Understand Closures

👉 Predict the output and explain how closure works here:

function outer() {
  let count = 0;
  return function inner() {
    count++;
    console.log(count);
  };
}

const counter = outer();
counter();
counter();
counter();

✅ Output:

1
2
3

🧠 Explanation:

The inner function remembers (closes over) the count variable from its lexical scope.

Each call increments the same count, not a new one.

4️⃣ (Predict Output) Function Hoisting inside another function

👉 Predict the output:

function test() {
  console.log(a);
  var a = 5;
  function a() {}
  console.log(a);
}
test();

✅ Output:

[Function: a]
5

🧠 Explanation:

Within test, the function a is hoisted before the variable.

After the assignment a = 5, it overwrites the function reference.

5️⃣ (Predict Output) this Binding in Different Contexts

👉 What will be logged?

const obj = {
  value: 42,
  getValue: function() {
    return this.value;
  }
};

const fn = obj.getValue;
console.log(fn());
console.log(obj.getValue());

✅ Output:

undefined
42

🧠 Explanation:

When fn() is called alone, this is undefined (in strict mode).

When obj.getValue() is called, this = obj.

6️⃣ (Coding) Build a Counter using Closure

👉 Write a function that keeps its own private counter and has methods to increment and decrement.

✅ Solution:

function createCounter() {
  let count = 0;
  return {
    increment: function() { 
	    count += 1;
	    return count;
     },
    decrement: function() { return --count; },
  };
}

const c = createCounter();
console.log(c.increment()); // 1
console.log(c.increment()); // 2
console.log(c.decrement()); // 1

🧠 Concept: Encapsulation via closures — count is private.

7️⃣ (Coding) Recursive Function – Factorial

👉 Write a recursive function that returns the factorial of a number.

✅ Solution:

function factorial(n) {
  if (n === 0) return 1;
  return n * factorial(n - 1);
}
console.log(factorial(5)); // 120

🧠 Concept: Each call creates a new execution context until the base case is reached.

8️⃣ (Coding) Fix this with .bind()

👉 The first greetFn() call prints undefined. Fix it so it prints the name.

const person = {
  name: "Jay",
  greet: function(city) {
    console.log(`Hi, I’m ${this.name} from ${city}`);
  }
};

const greetFn = person.greet;
greetFn("Delhi");                // ❌ undefined
const boundGreet = greetFn.bind(person);
boundGreet("Ahmedabad");         // ✅ Hi, I’m Jay from Ahmedabad

🧠 Concept: .bind() returns a new function with this permanently set to the given object.

🔟 (DOM Task) Closure inside Event Handler

👉 Use a closure so each list item alerts its own name when clicked.

✅ Solution:

<ul id="menu"></ul>
<script>
const items = ["Home", "About", "Contact"];
const ul = document.getElementById("menu");

items.forEach((item, index) => {
  const li = document.createElement("li");
  li.textContent = item;
  li.addEventListener("click", () => {
    alert(`You clicked ${item} (index: ${index})`);
  });
  ul.appendChild(li);
});
</script>

🧠 Concept: Each callback captures its own item and index through closure, so values don’t get overwritten.
