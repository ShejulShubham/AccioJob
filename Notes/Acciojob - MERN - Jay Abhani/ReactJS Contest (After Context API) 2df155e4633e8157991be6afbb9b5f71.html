Section C – Coding Round (3 questions)

Coding Q1 – Smart Recipe Filter

Concepts: Components, props, list rendering, conditional rendering, useState

Build a RecipeList feature with the following behavior:

  • You have a static list of recipe objects:

    { id, name, difficulty: "easy" | "medium" | "hard", isVegetarian: boolean }

  • The UI should include:
    • A text input to filter recipes by name (case-insensitive).
    • Two toggles/checkboxes:
      • “Only vegetarian”
      • “Difficulty: hard only”
  • Render the filtered list as cards.
  • If no recipe matches, show: “No recipes found. Try relaxing the filters.”

Requirements:

  • Use at least two separate child components (e.g., FilterControls and RecipeListView).
  • Use useState to manage filters.
  • Use conditional rendering for the empty state message.

Coding Q2 – Auto-Save Note with Status Indicator

Concepts: useState, useEffect, side effects, cleanup

Create a NoteEditor component that simulates auto-save:

Behavior:

  • A multiline textarea for the note content.
  • Whenever the user stops typing for 1.5 seconds, the note is “auto-saved”.
  • Status text in a corner:
    • “Typing…” while the user is actively typing.
    • “Saving…” while a fake save request is in progress.
    • “All changes saved” after the fake request completes.

Implementation hints:

  • Simulate save with setTimeout inside useEffect (e.g., 1 second “network delay”).
  • Do not call a real API.

Requirements:

  • Use useState to track content and status.
  • Use useEffect with a cleanup function to handle the debounce timer correctly.
  • Avoid infinite re-render loops by choosing the dependency array carefully.

Coding Q3 – Mini Dashboard with Context + Routing

Concepts: Context API, React Router, useEffect, data fetching

Build a mini dashboard with two pages using React Router and Context:

  • Routes:
    • / – Overview page
    • /settings – Settings page
  • Create UserPreferencesContext with:

    { theme: "light" | "dark", username: string }

  • Place the Context Provider at the root (wrapping the router).

Overview page (/):

  • Show greeting using username from context.
  • Show current theme.
  • Button: “Toggle Theme” – toggles between "light" and "dark" in context.
  • On mount, simulate data fetching with useEffect and setTimeout to load a list of notifications.
    • While loading: show “Loading…”.
    • After loading: show the list or “No notifications.”

Settings page (/settings):

  • Input to change username (updates context).
  • Show preview text: Preview as {username} · Theme: {theme}.

Requirements:

  • Use React Router v6 style (e.g., BrowserRouter, Routes, Route, Link / useNavigate).
  • Use Context API with useContext to read and update preferences.
  • Use at least one useEffect for simulated data fetching.
