useMemo and useCallback

🧠 PART 1: React Re-rendering — The Real Truth

1️⃣ What does “re-render” ACTUALLY mean in React?

Re-render ≠ page refresh

👉 Re-render means:

React re-runs your component function to calculate new JSX.

That’s it.

Simple example

function App() {
  console.log("Component rendered");
  return <h1>Hello</h1>;
}

Whenever React decides to re-render:

  • App() function runs again
  • console.log runs again
  • JSX is recalculated

❌ Browser page does NOT refresh

❌ DOM is NOT rebuilt fully

✅ React only updates what changed

2️⃣ When does a component re-render?

A component re-renders when:

  1. State changes
  1. Props change
  1. Parent re-renders
  1. Context value changes

Example: State change

function Counter() {
  const [count, setCount] = React.useState(0);

  console.log("Counter rendered");

  return (
    <button onClick={() => setCount(count + 1)}>
      Count: {count}
    </button>
  );
}

🧠 Every click:

  • setCount → state changes
  • React re-runs Counter()
  • JSX recalculated

This is EXPECTED and CORRECT

3️⃣ Important rule (Most people miss this)

React re-runs the entire component function on every render

That means:

// ALL of this runs again
const value = heavyCalculation();
const arr = [];
const obj = {};
const fn = () => {};

Even if:

  • UI looks same
  • Logic didn’t change

React doesn’t “skip lines” inside component functions.

4️⃣ What is an “expensive computation”?

An expensive computation is:

  • Heavy loop
  • Large data processing
  • Filtering / sorting big arrays
  • Complex math
  • Anything that takes noticeable time

Example

function slowFunction(num) {
  console.log("Slow function running...");
  let result = 0;

  for (let i = 0; i < 1_000_000_000; i++) {
    result += num;
  }

  return result;
}

Now use it:

function App() {
  const [count, setCount] = React.useState(0);

  const result = slowFunction(count);

  return (
    <>
      <h1>{result}</h1>
      <button onClick={() => setCount(count + 1)}>
        Increase
      </button>
    </>
  );
}

😨 Problem:

  • Every click → component re-renders
  • slowFunction runs again
  • App feels laggy / frozen

5️⃣ The REAL problem (core issue)

React is FAST, but your code may not be

React is doing its job:

  • State changed → re-render → correct

But:

  • Some calculations don’t need to run again
  • Some functions don’t need to be recreated
  • Some values did not logically change

Yet React has no memory by default

6️⃣ Why React doesn’t cache by default?

Because:

  • React components are pure functions
  • Given same state + props → same UI
  • React avoids assumptions

So React says:

“I’ll rerun everything. YOU tell me what to cache.”

That’s where:

  • useMemo
  • useCallback

come in.

7️⃣ Key takeaway from Part 1 (VERY IMPORTANT)

📌 Re-render means function re-execution

📌 Every variable inside component is recreated

📌 Expensive logic runs again unless optimized

📌 React doesn’t cache values or functions automatically

🔑 Mental Model (Lock this in your head)

State change
   ↓
Component function runs again
   ↓
All variables recreated
   ↓
All functions recreated
   ↓
All calculations rerun

❗ Common Beginner Misunderstanding

❌ “React updates only changed lines”

❌ “Only JSX re-renders”

✅ Entire component function runs again

🧠 PART 2: Reference Equality (The REAL Problem)

❗ Almost all useMemo / useCallback confusion comes from not understanding this

1️⃣ Primitive vs Reference values (VERY BASIC, VERY IMPORTANT)

🔹 Primitive values

number, string, boolean, null, undefined, symbol

Example:

let a = 10;
let b = 10;

console.log(a === b); // true

👉 React compares value

🔹 Reference values

object, array, function

Example:

let a = {};
let b = {};

console.log(a === b); // false ❌

Why?

  • Because they live at different memory locations

Even though:

{} and {}

look same, they are not the same reference

2️⃣ What happens inside a React component?

Example

function App() {
  const arr = [];
  const obj = {};
  const fn = () => {};

  console.log(arr === arr); // true (same reference)
  return null;
}

Now important part:

👉 On EVERY re-render, React re-runs this function.

So this happens:

Render 1 → arr #1, obj #1, fn #1
Render 2 → arr #2, obj #2, fn #2
Render 3 → arr #3, obj #3, fn #3

Even though code is same, references are new every time.

3️⃣ Why React thinks “something changed” (even when it didn’t)

React compares reference, not deep values.

Example

function App() {
  const [count, setCount] = React.useState(0);

  const user = { name: "Jay" };

  return (
    <>
      <button onClick={() => setCount(count + 1)}>
        Increase
      </button>
      <Child user={user} />
    </>
  );
}

What React sees:

  • count changes → parent re-renders
  • user is recreated
  • New object reference
  • React thinks: ❗ “Props changed”

So:

👉 Child re-renders unnecessarily

4️⃣ Functions are ALSO reference values (BIGGEST TRAP)

Example

function App() {
  const handleClick = () => {
    console.log("Clicked");
  };

  return <Child onClick={handleClick} />;
}

On every render:

() => {} !== () => {}

So:

  • New function reference
  • Child thinks props changed
  • Child re-renders

Even though logic is same 😐

5️⃣ Real-world bug example (VERY COMMON)

Parent

function Parent() {
  const [count, setCount] = React.useState(0);

  const handleClick = () => {
    console.log("Hello");
  };

  return (
    <>
      <button onClick={() => setCount(count + 1)}>+</button>
      <Child onClick={handleClick} />
    </>
  );
}

Child

const Child = React.memo(({ onClick }) => {
  console.log("Child rendered");
  return <button onClick={onClick}>Child Button</button>;
});

❌ EXPECTATION:

Child should not re-render when count changes

❌ REALITY:

Child re-renders every time

WHY?

👉 Because:

handleClick !== previous handleClick

6️⃣ Reference Equality Mental Model (LOCK THIS)

Primitive

10 === 10  ✅
"hi" === "hi" ✅

Reference

{} === {} ❌
[] === [] ❌
() => {} === () => {} ❌

7️⃣ This is the CORE problem useMemo/useCallback solve

React asks you:

“Should I reuse the old reference or create a new one?”

Solutions:

ProblemSolution
Expensive value recalculateduseMemo
Function recreateduseCallback

8️⃣ Key Takeaways (Very Important)

📌 Objects, arrays, functions are new on every render

📌 React compares references, not deep equality

📌 New reference = React thinks “changed”

📌 This causes unnecessary re-renders

📌 useMemo & useCallback exist ONLY to solve this

❗ DO NOT MEMORIZE THIS LINE — UNDERSTAND IT

useMemo caches values

useCallback caches functions
