ReactJS Cheatsheet

⚛️ React Interview Cheat Sheet (1-Page)

🔹 What is React?

  • JavaScript library for building UI
  • Component-based architecture
  • Declarative UI
  • Uses Virtual DOM for performance

🔹 JSX

  • JavaScript + HTML syntax
  • Transpiled by Babel to React.createElement
  • JSX is optional

🔹 Components

  • Functional Components (preferred)
  • Class Components (legacy)
function Component() {
  return <div>Hello</div>
}

🔹 Props vs State

PropsState
Passed from parentManaged inside component
Read-onlyMutable
Makes components reusableTriggers re-render

🔹 Virtual DOM & Rendering

  • Virtual DOM = lightweight JS copy of real DOM
  • React uses diffing + reconciliation
  • Only changed nodes update in real DOM
  • Page does NOT refresh

🔹 Hooks (Most Important)

Common hooks:

  • useState
  • useEffect
  • useContext
  • useRef
  • useMemo
  • useCallback

Rules of Hooks

  • Call at top level only
  • Call inside functional components or custom hooks

🔹 useState

const [count, setCount] = useState(0)
  • Async updates
  • Re-render on change
  • Lazy init: useState(() => expensiveFn())

🔹 useEffect

useEffect(() => {
  // side effect
  return () => cleanup()
}, [deps])
DependencyRuns When
noneEvery render
[]On mount
[x]When x changes

🔹 Context API

  • Solves props drilling
  • For global state (auth, theme)
const MyContext = createContext()

⚠️ Context re-renders all consumers on value change

🔹 Performance Optimization

  • React.memo → memoize components
  • useMemo → memoize values
  • useCallback → memoize functions
  • Avoid inline functions & objects
  • Use proper key in lists

🔹 Routing (React Router)

Key components:

  • BrowserRouter
  • Routes
  • Route
  • Link
  • useParams
  • useNavigate

Link ≠ <a> (no page reload)

🔹 Forms

  • Controlled components (recommended)
  • Input value comes from state
  • Use event.preventDefault()

🔹 useRef

Used for:

  • DOM access
  • Focus handling
  • Storing mutable values (no re-render)

🔹 Error Boundaries

  • Catch runtime UI errors
  • Must be class components
  • Do NOT catch:
    • Event handler errors
    • Async errors

🔹 Testing

Tools:

  • Jest
  • React Testing Library
  • Cypress (E2E)

Test behavior, not implementation

🔹 Advanced Concepts

  • SSR & Hydration
  • Code splitting (React.lazy)
  • Concurrent features
  • useTransition
  • useDeferredValue

🔹 Common Interview Mistakes

❌ Missing dependency array

❌ Using index as key

❌ Overusing Context

❌ Unnecessary state

❌ No memoization

🔹 Final Interview Tip

Explain WHY, not just HOW.

Give real examples from projects.
