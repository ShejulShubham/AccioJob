Quick Notes Board

Mini Project: Quick Notes Board (DOM Edition)

Goal: Add, edit, delete, and filter simple notes using vanilla DOM only.

Students will practice: getElementById, querySelector, events (submit, click, input), createElement, appendChild, remove, textContent, basic state, and re-rendering.

Quick Notes Board — Hints (Step-by-Step)

0) Mental model (before you code)

  • Treat the array notes as the single source of truth (each note = {id, title, text}).
  • The screen should always reflect what’s in that array.

    👉 Any time the data changes (add/edit/delete/filter), re-render the list from the array.

1) DOM basics to set up

  • Cache all important elements once at the top:
    • Form, inputs (title, text), submit button, cancel button
    • Hidden input for editingId
    • Notes container (#notes)
    • Filter input and error message node
  • Add event listeners:
    • form.submit → add or save changes
    • cancel-edit.click → exit edit mode
    • filter.input → update search text and re-render
    • Delegation on #notes for clicks on Edit/Delete (one listener, not one per card)

2) State you’ll keep

  • notes (array of note objects)
  • filterText (string, default "")
  • You don’t need complex state; the hidden input editingId can tell you whether you’re in add mode (empty value) or edit mode (has an id).

3) Rendering flow (core engine)

  • Always start a render by clearing the container (notesEl.innerHTML = "").
  • Create a visible list:
    • Convert filterText to lowercase.
    • Include a note if its title or text contains the query.
  • If visible is empty, show a small “No notes to show.” message and return early.
  • For each note in visible, build elements with createElement:
    • <article class="note" data-id="...">

      → <h3> title

      → <p> text

      → <div class="note-actions"> with two <button>s: Edit, Delete

  • Append each card to the container.
Hint: Keep the render function pure (no side effects other than DOM creation). It should only read from notes and filterText, and then draw.

4) Add vs Edit (one form, two modes)

Detect mode on submit:

  • If hidden editingId is empty → Add mode
  • If it has a value → Edit mode

Add mode (high level):

  1. Trim and validate the title. If empty → show error and stop.
  1. Make a new note object (unique id via Date.now().toString() is fine).
  1. Push it into notes.
  1. Re-render, reset the form, and ensure you’re back to “Add Note” UI.

Edit mode (high level):

  1. Trim and validate the title. If empty → show error and stop.
  1. Find the note in the array with that id.
  1. Update its title and text.
  1. Re-render.
  1. Clear editingId, change submit button text back to “Add Note”, hide Cancel Edit, reset the form.
Hint: Keep all edits inside the array first; the UI will update on the next render.

5) Entering Edit mode (when Edit button is clicked)

  • Use event delegation on #notes:
    • Find if the click target has class edit-btn.
    • Get the closest .note and read its data-id.
    • Find that note in notes.
    • Pre-fill the form fields with that note’s data.
    • Put the id into the hidden editingId.
    • Change submit button text to “Save Changes”.
    • Show the Cancel Edit button.
    • (Optional) focus the title input so students can type immediately.

6) Deleting a note (when Delete button is clicked)

  • Still under the same delegated click handler:
    • If target has class delete-btn, grab the data-id.
    • Remove that item from the notes array (find index → splice).
    • If you were editing this same note, call the cancel edit flow first so the form resets properly.
    • Re-render.
Hint: Removing from the array is what truly deletes the note. The DOM will reflect the new array on re-render.

7) Filter logic (why other notes don’t “disappear” for real)

  • Filtering should not modify the main notes array.
  • It only creates a temporary visible array used for displaying.
  • When filter is cleared, the next render just shows everything again.
Hint: If it feels like items “disappear”, that’s good—they’re just not in the visible set. Your data is safe.

8) Validation (keep it minimal)

  • Only the title is required.
  • On invalid title:
    • Put a small message ("Title is required.") into the error node.
    • Return early (don’t add or save).
  • On successful submit:
    • Clear the error node.
Hint: Always trim() inputs before checking emptiness.

9) Empty state & polish

  • If no notes match the filter or the list is empty, show a small muted paragraph.
  • Keep button texts accurate to the mode:
    • Add mode → “Add Note”
    • Edit mode → “Save Changes”
  • The Cancel Edit button should only be visible in edit mode.

10) Event Delegation (the pattern)

  • Attach one click listener to #notes.
  • Use event.target.classList.contains("edit-btn") (or delete-btn) to detect which action.
  • Use closest(".note") to get the parent card and its data-id.
Hint: This avoids attaching many listeners to each card. New cards work automatically.

11) Testing checklist (quick manual tests)

  1. Add two different notes → both appear.
  1. Filter by a word present in only one note → only that note shows.
  1. Clear filter → both notes show again.
  1. Click Edit on the first note → form pre-fills; change values; Save Changes → card updates.
  1. Click Cancel Edit midway → form clears and returns to Add mode.
  1. Delete a note → it disappears; other notes remain.
  1. Try to submit empty title → error shows; nothing adds/updates.

12) Common bugs & quick fixes

  • Nothing shows after filtering: check lowercase conversion and includes()/indexOf() logic.
  • Edit doesn’t update: ensure you’re finding the note by the same id you set in editingId.
  • Stuck in edit mode: remember to clear editingId, change button text, and hide Cancel.
  • Delete doesn’t work: confirm you’re reading data-id from the clicked card (not from the wrong element).
  • Render duplicates: always clear #notes before drawing.
  • Validation ignored: return early after setting the error message.

13) Light UX hints (still basic)

  • Focus the title input when entering edit mode (helps speed).
  • Keep messages short and visible near the field.
  • Keep the layout simple—don’t overthink styles; this is a DOM task.

Mini pseudo-flow you can keep on a sticky note

  • On load → render()
  • Submit:
    • if editingId → update note → reset to add mode → render()
    • else → add new note → render() → reset form
  • Click in board:
    • Edit → prefill + switch to edit mode
    • Delete → remove from array → render()
  • Filter input:
    • update filterText → render()

var formEl = document.getElementById("note-form");
var titleInput = document.getElementById("title");
var textInput = document.getElementById("text");
var submitBtn = document.getElementById("submit-btn");
var cancelEditBtn = document.getElementById("cancel-edit");
var editingIdInput = document.getElementById("editing-id");
var titleErrorEl = document.getElementById("title-error");
var filterInput = document.getElementById("filter");

// ------------------------
// Init
// ------------------------
function init() {
  // TODO(1): Render notes from the current state (notes array).
  renderNotes();

  // Events
  formEl.addEventListener("submit", onSubmit);
  cancelEditBtn.addEventListener("click", onCancelEdit);
  filterInput.addEventListener("input", onFilterChange);

  // TODO(2): Add event delegation on the notes container for Edit/Delete buttons.
  // Example: notesEl.addEventListener("click", onNotesClick);
  notesEl.addEventListener("click", onNotesClick);

// ------------------------
// Rendering
// ------------------------
function renderNotes() {
  // TODO(3): Clear notesEl content before re-rendering.
  notesEl.innerHTML = "";

  // TODO(4): Compute filtered list from notes array using filterText.
  var q = (filterText || "").toLowerCase();
  var visible = notes.filter(function (n) {
    var t = (n.title || "").toLowerCase();
    var x = (n.text || "").toLowerCase();
    return t.indexOf(q) !== -1 || x.indexOf(q) !== -1;
  });

  // TODO(5): If no visible notes, show a <p class="muted"> message and return.
  if (visible.length === 0) {
    var emptyP = document.createElement("p");
    emptyP.className = "muted";
    emptyP.textContent = "No notes to show.";
    notesEl.appendChild(emptyP);
    return;
  }

  // TODO(6): For each visible note, build the DOM structure using createElement:
  // - <article class="note" data-id="...">
  //   - <h3>Title here</h3>
  //   - <p>Text here</p>
  //   - <div class="note-actions">
  //       <button class="edit-btn">Edit</button>
  //       <button class="delete-btn">Delete</button>
  //     </div>
  // Append each built card to notesEl.
  for (var i = 0; i < visible.length; i++) {
    var n = visible[i];

    var card = document.createElement("article");
    card.className = "note";
    card.setAttribute("data-id", n.id);

    var h3 = document.createElement("h3");
    h3.textContent = n.title;

    var p = document.createElement("p");
    p.textContent = n.text;

    var actions = document.createElement("div");
    actions.className = "note-actions";

    var editBtn = document.createElement("button");
    editBtn.className = "edit-btn";
    editBtn.type = "button";
    editBtn.textContent = "Edit";

    var delBtn = document.createElement("button");
    delBtn.className = "delete-btn";
    delBtn.type = "button";
    delBtn.textContent = "Delete";

    actions.appendChild(editBtn);
    actions.appendChild(delBtn);

    card.appendChild(h3);
    card.appendChild(p);
    card.appendChild(actions);

    notesEl.appendChild(card);
  }
}

// ------------------------
// Events
// ------------------------
function onSubmit(e) {
  e.preventDefault();
  clearError();

  var title = titleInput.value.trim();
  var text = textInput.value.trim();
  var editingId = editingIdInput.value;

  // TODO(7): Validate that title is not empty.
  // - If empty, set titleErrorEl.textContent with a message and return.
  if (title === "") {
    titleErrorEl.textContent = "Title is required.";
    return;
  }

  if (editingId) {
    // EDIT MODE
    // TODO(8): Find the note by id and update its title and text.
    var found = null;
    for (var i = 0; i < notes.length; i++) {
      if (notes[i].id === editingId) {
        found = notes[i];
        break;
      }
    }
    if (found) {
      found.title = title;
      found.text = text;
    }

    // TODO(9): Reset the form back to "Add Note" mode (editingIdInput, buttons, labels).
    editingIdInput.value = "";
    submitBtn.textContent = "Add Note";
    cancelEditBtn.classList.add("hidden");
  } else {
    // ADD MODE
    // TODO(10): Create a new note object with a unique id (e.g., Date.now().toString()).
    var newNote = {
      id: Date.now().toString(),
      title: title,
      text: text
    };
    // TODO(11): Push the new note into the notes array.
    notes.push(newNote);
  }

  // TODO(12): Re-render notes and reset the form fields.
  renderNotes();
  formEl.reset();
}

function onCancelEdit() {
  // TODO(13): Clear edit state and reset form to "Add Note" mode.
  // - editingIdInput.value = ""
  // - submitBtn text back to "Add Note"
  // - hide cancelEditBtn
  // - clear any error
  formEl.reset();
  editingIdInput.value = "";
  submitBtn.textContent = "Add Note";
  cancelEditBtn.classList.add("hidden");
  clearError();
}

function onFilterChange(e) {
  // TODO(14): Update filterText from the input value and re-render notes.
  filterText = e.target.value || "";
  renderNotes();
}

// TODO(15): Implement delegated click handling for Edit/Delete actions on the notes board.
function onNotesClick(e) {
  // - Determine if the click was on .edit-btn or .delete-btn
  // - Find the closest .note element and read its data-id
  // - For Edit:
  //   - Find the note by id
  //   - Fill form inputs with the note values
  //   - Set editingIdInput.value = id
  //   - Change submit button text to "Save Changes"
  //   - Show cancelEditBtn
  // - For Delete:
  //   - Remove the note from the array by id
  //   - Re-render notes
  var target = e.target;
  var isEdit = target.classList.contains("edit-btn");
  var isDelete = target.classList.contains("delete-btn");
  if (!isEdit && !isDelete) return;

  var card = target.closest(".note");
  if (!card) return;

  var id = card.getAttribute("data-id");

  if (isEdit) {
    var note = null;
    for (var i = 0; i < notes.length; i++) {
      if (notes[i].id === id) {
        note = notes[i];
        break;
      }
    }
    if (!note) return;

    titleInput.value = note.title;
    textInput.value = note.text;
    editingIdInput.value = id;
    submitBtn.textContent = "Save Changes";
    cancelEditBtn.classList.remove("hidden");
    titleInput.focus();
  }

  if (isDelete) {
    // Remove note by id
    var idx = -1;
    for (var j = 0; j < notes.length; j++) {
      if (notes[j].id === id) {
        idx = j;
        break;
      }
    }
    if (idx !== -1) {
      notes.splice(idx, 1);
      // If we were editing this note, exit edit mode
      if (editingIdInput.value === id) {
        onCancelEdit();
      }
      renderNotes();
    }
  }
}

// Helpers
// ------------------------
function clearError() {
  // TODO(16): Clear any validation messages (e.g., titleErrorEl.textContent = "")
  titleErrorEl.textContent = "";
}
