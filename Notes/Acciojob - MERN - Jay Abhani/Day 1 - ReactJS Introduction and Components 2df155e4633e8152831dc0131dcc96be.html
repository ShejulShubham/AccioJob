Day 1 - ReactJS Introduction and Components

🔥 Topic 1: ReactJS Introduction

1. Why React Exists (The Real Developer Problem)

Data <→ UI

Earlier, we built websites by:

  • Writing HTML → static UI
  • Adding CSS → styling
  • Adding JavaScript → DOM manipulation

It worked, until our apps became interactive.

Imagine you're building:

  • Instagram Feed
  • Flipkart Product Page
  • YouTube Comments Section
  • Swiggy Cart Update System

These UIs change constantly based on user actions and data.

📌 The Old Way: Painful DOM Manipulation

Example: Updating a cart count on a website.

document.getElementById("cart-count").innerText = count;

Easy… until the UI grows big.

Then you do this in 10–20 places → bugs everywhere 😭

You update UI manually → repetitive & error-prone

Logic and UI mixed → messy

Scaling becomes impossible.

This is why developers needed something better.

2. What React Solves

React gives us:

✅ A way to build UI using components (reusable, modular code)

✅ A virtual DOM to update UI FAST

✅ A predictable way to manage state

✅ Easier to maintain large-scale apps

React focuses only on UI.

It doesn’t care about routing, data fetching, etc. (we add libraries later).

3. What is React? (Simple Definition)

React is a JavaScript library for building interactive user interfaces.

Created by Meta (Facebook) in 2013.

Used by:

  • Netflix
  • Amazon
  • Swiggy
  • Zomato
  • Uber
  • Instagram

The reason?

Because React lets you build UI using tiny pieces called components, and React automatically updates the UI when data changes.

4. Real-World Example: UI as LEGO Blocks

Think of UI like LEGO.

You don’t build a house in one piece.

You build it using small blocks:

  • Header Block
  • Navbar Block
  • Product Card Block
  • Footer Block

React components work exactly like LEGO.

5. Example: Non-React vs React Thinking

🧡 Non-React Approach

You manually edit DOM:

document.querySelector("#btn").addEventListener("click", () => {
  count++;
  document.getElementById("counter").innerText = count;
});

You control everything.

💙 React Approach

You tell React:

“If count changes, update the UI automatically.”

React handles DOM updates for you.

6. Why Companies Prefer React

🔹 Reusable components

Build a ProductCard once → use it 50 times.

🔹 Fast rendering (Virtual DOM)

React calculates the smallest possible update and applies it.

🔹 Huge ecosystem

Thousands of libraries, tools, community support.

🔹 Easy to maintain

Code is structured, modular, and predictable.

7. What React Is NOT

React is NOT:

  • A framework
  • A full-stack solution
  • A backend tool
  • A replacement for Node.js

React is only about building the UI layer.

8. Example UI Breakdown (React Thinking)

Imagine Swiggy's restaurant page:

  • RestaurantHeader component
  • MenuList component
  • MenuItem component
  • CartSummary component

Each one is a component that can be reused, rendered, hidden, or updated independently.

⭐ 1. JSX Must Return One Parent Element

This WILL confuse beginners, so explain simply:

Wrong JSX:

return (
  <h1>Hello</h1>
  <p>Welcome</p>
);

Correct way (wrap in a div):

return (
  <div>
    <h1>Hello</h1>
    <p>Welcome</p>
  </div>
);

Or use Fragment:

return (
  <>
    <h1>Hello</h1>
    <p>Welcome</p>
  </>
);

📌 JSX must return one parent — otherwise React won't understand where the component starts/ends.

⭐ 2. JSX Expressions: Insert any JS using { }

Example:

const name = "Ujjwal";

return (
  <h1>Welcome, {name}</h1>
);

Inside {} you can put:

  • Variables
  • Function calls
  • Math
  • JS expressions

Example:

<h2>2 + 2 = {2 + 2}</h2>

Example:

<p>Message length: {message.length}</p>

⭐ 3. JSX Attributes (NOT HTML attributes)

JSX uses camelCase, not HTML names.

HTMLJSX
classclassName
forhtmlFor
onclickonClick
tabindextabIndex

Example:

<div className="box">Hello</div>

Another example:

<label htmlFor="email">Email</label>

⭐ 4. Inline Styling in JSX

HTML way (not allowed in React):

<div style="color: red; background: yellow"></div>

React way:

<div style={{ color: "red", backgroundColor: "yellow" }}>
  Styled Box
</div>

Two curly braces:

  • outer {} = JSX expression
  • inner {} = JavaScript object

⭐ 5. JSX Cannot Have If Statements Directly (Common mistake)

❌ Wrong:

return (
  if(show) {
    <h1>Hello</h1>
  }
)

✔ Right — use ternary:

return (
  show ? <h1>Hello</h1> : <p>Hidden</p>
);

✔ Or use logical AND:

return (
  show && <h1>Hello</h1>
);

⭐ 6. JSX Self-Closing Tags

Just like HTML, but more strict.

✔ Correct:

<img src="pic.jpg" />
<input />
<br />

❌ Wrong:

<img></img>

⭐ 7. You Can Write Functions Inside JSX

Example:

function getGreeting() {
  return "Good Evening";
}

return (
  <h1>{getGreeting()}</h1>
);

Example with mapping:

const fruits = ["Apple", "Banana", "Grapes"];

return (
  <ul>
    {fruits.map(item => <li>{item}</li>)}
  </ul>
);

⭐ 8. JSX Must Return a Value

All React components MUST return something.

✔ Works:

return <h1>Hello</h1>;

❌ Doesn't work:

return
  <h1>Hello</h1>;

Because JavaScript inserts a semicolon after return → React returns nothing.

Teach students to write:

return (
  <h1>Hello</h1>
);

clock using pure-react

<!DOCTYPE html>
<html lang="en">

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Current Time</title>
</head>

<body>
    <div id="root"></div>

    <script src="https://unpkg.com/react@18/umd/react.development.js"></script>
    <script src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script>

    <script>
        let root = ReactDOM.createRoot(document.getElementById("root"));

        function showTime() {
            let timeNow = new Date().toLocaleTimeString();

            let element = React.createElement("h1", null, timeNow);

            root.render(element);
        }

        setInterval(showTime, 1000);
    </script>
</body>

</html>
